Content Creation & Composer

Rich text editing, media attachments, polls, space selection, and post submission

Key files: app/create-post.tsx (screen), components/composer/CreatePostContent.tsx (main component)
The composer uses 10tap editor for rich text, supports images (up to 4), video embeds, GIFs, and polls. Posts are created and edited through the same screen.

1. Composer Overview

The composer screen (/create-post) handles both creating new posts and editing existing ones. It's presented as a full-screen modal that slides up from the bottom.

3. Rich Text Editing (10tap)

The editor uses @10play/tentap-editor, a WebView-based rich text editor that outputs HTML.

Editor Setup

hooks/useThemedEditor.ts configures the editor bridge with theme-aware styling:

const editor = useEditorBridge({
  autofocus: false,
  avoidIosKeyboard: true,
  bridgeExtensions: [...TenTapStartKit],
  theme: {
    webview: { backgroundColor: colors.background },
  },
});

// Apply theme colors via CSS injection
useEffect(() => {
  editor.injectCSS(`
    body { color: ${colors.text}; }
    a { color: ${colors.primary}; }
    /* ... more theme CSS ... */
  `);
}, [colors]);
Never use PlaceholderBridge.configureExtension(). It silently crashes the WebView. TenTapStartKit already includes PlaceholderBridge. Set the placeholder at runtime with editor.setPlaceholder(text) instead.

Dark Mode

The editor requires both a theme.webview.backgroundColor setting and CSS injection to match the app theme. Without the CSS injection, text and links won't match the app's dark mode colors.

5. HTML-to-Markdown Conversion

utils/htmlToMarkdown.ts converts the 10tap editor's HTML output to markdown for server submission:

  • Custom regex-based converter — no external dependencies (RN has no DOM, so turndown won't work)
  • Handles only 10tap's limited HTML subset (not arbitrary HTML)
  • Preserves code blocks by protecting them before other conversions
  • Converts headings, lists, blockquotes, links, inline formatting
  • Decodes HTML entities
  • Cleans up whitespace (collapses 3+ consecutive newlines to 2)
10tap's HTML subset only. This converter handles the limited tags 10tap produces (<h2>, <strong>, <em>, <ul>, <ol>, <blockquote>, <code>, <a>). It's not a general-purpose HTML-to-markdown converter.

6. Attachment Types

The composer supports four attachment types. Only one of video/GIF can be active at a time, but images can be combined with a poll.

Images

  • Picker: expo-image-picker with multi-selection (up to 4)
  • Upload: Each image uploaded via uploadMedia(uri, type, fileName, 'feed')
  • Preview: components/composer/MediaPreview.tsx shows horizontal thumbnail strip with remove buttons
  • Quality: 0.8 (80% JPEG compression)

Video Embed

  • Input: components/composer/VideoAttachModal.tsx — bottom sheet with URL input
  • Validation: Fetches oembed data via GET /feeds/oembed?url={url}
  • Preview: components/composer/VideoPreview.tsx shows thumbnail + title + provider
  • Supports: YouTube, Vimeo, Wistia, and other oembed providers

GIF

  • Picker: components/composer/GifPickerModal.tsx — Giphy search with 2-column grid
  • Preview: components/composer/GifPreview.tsx shows animated preview with remove button
  • API: searchGifs(query, offset) from services/api/giphy.ts

ComposerToolbar

components/composer/ComposerToolbar.tsx renders the bottom action bar:

7. Polls & Surveys

components/composer/PollBuilderSheet.tsx provides poll creation in a bottom sheet:

Poll Configuration

SettingOptions
Typesingle_choice or multi_choice (toggle switch)
Options2–4 text options (minimum 2 must be filled)
End dateNot yet implemented in UI

Poll Preview

components/composer/PollPreview.tsx shows a card with option count and preview text. Tappable to edit, with a remove button.

Submission Format

// Survey data in post payload
survey: {
  type: 'single_choice',  // or 'multi_choice'
  options: [
    { label: 'Option 1', slug: 'opt_1' },
    { label: 'Option 2', slug: 'opt_2' },
  ],
  end_date: '',
}
Slugs are auto-generated. Option slugs follow the pattern opt_1, opt_2, etc. They're generated at submission time, not entered by the user. The server stores the poll configuration in meta.survey_config.

8. Space Selection

components/composer/SpaceSelector.tsx lets users choose which space to post in:

  • Trigger: Button showing selected space name or "Select Space" placeholder
  • Modal: Searchable list grouped by space groups (hierarchical)
  • Data: Fetches from spacesApi.getSpaces() + spacesApi.getSpaceGroups()
  • Search: Filters by space title or slug
  • Privacy icons: Lock (secret), people (private), globe (public)
  • Output: Returns space slug + title to parent component
Always send space slug, not ID. The Fluent Community API expects the space slug (e.g., 'general') in the space field, not the numeric ID. This matches the web app behavior.

The space selector is hidden in edit mode (you can't move a post to a different space) and when a spaceSlug param is already provided.

9. Post Submission API

services/api/feeds.ts → createFeed(data)

Request Payload

POST /feeds

{
  type: 'text',
  message: 'Markdown content here',    // Required
  space: 'general',                     // Space slug (required)
  content_type: 'markdown',             // or 'text', 'html', 'survey'
  title: 'Optional Title',
  privacy: 'public',
  status: 'published',

  // Images (up to 4)
  media_images: [
    { url: '...', type: 'image/jpeg', width: 800, height: 600, provider: 'uploader' }
  ],

  // Video embed (one)
  media: { type: 'oembed', url: 'https://youtube.com/watch?v=...' },

  // GIF
  meta: {
    media_preview: {
      image: 'https://giphy.com/...',
      type: 'image',
      provider: 'inline',
      width: 480,
      height: 270,
    }
  },

  // Poll
  survey: {
    type: 'single_choice',
    options: [{ label: 'Yes', slug: 'opt_1' }, { label: 'No', slug: 'opt_2' }],
    end_date: '',
  },

  // Always included (defaults)
  media_image: '',
  topic_ids: [],
  send_announcement_email: 'no',
}

Response

{
  success: true,
  data: { feed: Feed }  // The created feed object
}

After Submission

On success, the composer emits CACHE_EVENTS.FEEDS to refresh the feed list, then navigates back. The new post appears at the top of the relevant feed.

10. Post Editing

Edit mode is triggered by passing editId to the create-post screen. The existing post is fetched and all fields are populated.

Edit Flow

  1. Screen receives editId route param
  2. Fetches post via feedsApi.getFeedById(editId)
  3. Populates editor with existing content:
    • Title and message body loaded into editor
    • Existing images re-attached from meta.media_items
    • Video/GIF previews restored from meta.media_preview
    • Poll options extracted from meta.survey_config
  4. Submit button label changes to "Save"
  5. Space selector is hidden (can't change space)
  6. On save: calls feedsApi.updateFeed(id, data)
Same component, different mode. The composer doesn't have separate create/edit components. It's the same CreatePostContent with conditional behavior based on whether editId is present.

11. Keyboard Handling

The composer uses react-native-keyboard-controller with specific layout patterns:

  • KAV: KeyboardAvoidingView with behavior="padding"
  • Toolbar position: Absolutely positioned at bottom with bottom: insets.bottom
  • Media previews: Rendered inside the KAV, above the keyboard
Toolbar uses absolute positioning exception. For the composer toolbar, safe area padding is applied directly on the KAV style as bottom: insets.bottom, NOT as a sibling spacer. This is the exception to the normal KAV pattern (see Architecture). The toolbar is absolutely positioned, so the sibling spacer pattern doesn't apply here.

13. Rules

  1. Always send content_type: 'markdown' for regular posts. The 10tap editor produces HTML, which htmlToMarkdown() converts before submission. The server expects markdown. Only use 'survey' when a poll is attached.
  2. Never use PlaceholderBridge.configureExtension(). It crashes the 10tap WebView. Use editor.setPlaceholder(text) at runtime instead. TenTapStartKit already includes PlaceholderBridge.
  3. Send space slug, not ID. The space field in the API payload is the space's URL slug (e.g., 'general'), not its numeric database ID.
  4. Images go in media_images, videos in media, GIFs in meta.media_preview. These are three different fields with different structures. Don't mix them up.
  5. Poll option slugs are auto-generated. Use the opt_1, opt_2 pattern. Don't let users set custom slugs. The server uses these for vote tracking.
  6. Max 4 images per post. The picker's selectionLimit accounts for already-attached images. This limit matches the Fluent Community web UI.
  7. Use useThemedEditor() for all 10tap instances. The hook handles theme CSS injection and dark mode. Don't set up the editor bridge manually. It's shared by the composer, comment sheet, and blog comments.
  8. Emit CACHE_EVENTS.FEEDS after successful create/update. This triggers feed list refresh so the new or edited post appears immediately. Don't navigate back without emitting.
  9. htmlToMarkdown() handles only 10tap's HTML subset. It's not a general-purpose converter. Don't use it for arbitrary HTML from the server or external sources.
  10. Server enforces posting permissions. The composer doesn't pre-check whether the user can post in a space. If the user lacks permission, the API returns 403 and the composer shows an error. Don't add client-side permission checks that could get out of sync with the server.
