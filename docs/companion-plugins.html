2. TBC Community App

companion plugins/tbc-community-app/ Core

The main bridge plugin. Every app installation requires this. It provides JWT authentication, registration, the batch API, push notifications, app configuration delivery, and authenticated WebView sessions.

3. TBC Multi Reactions

companion plugins/tbc-multi-reactions/ Core

Replaces Fluent Community’s default single-heart reaction with a multi-emoji reaction system. Supports posts, comments, and chat messages. Reaction icons can be customized in WP admin (PNG, SVG, GIF, WEBP).

Endpoints

MethodEndpointAuthPurpose
GET/configNoGet enabled reaction types and icon URLs
POST/swapYesSwap reaction type on an existing reaction
GET/breakdown/{type}/{id}NoGet reaction counts by type for a post/comment
GET/breakdown/{type}/{id}/usersNoGet reaction breakdown with user details

How It Works

  • Injects reaction data into Fluent Community’s standard feed/comment API responses (no extra API calls needed for basic display)
  • The /config endpoint provides icon URLs so the app can display custom reaction graphics
  • The /swap endpoint lets users change their reaction type without removing and re-adding
  • Controlled by the multi_reactions feature flag (server-controlled via wp-admin → TBC Community App → Features tab, accessed via useFeatures()) — falls back to default heart if disabled

4. TBC Starter Theme

companion plugins/tbc-starter-theme/ Core

A lightweight WordPress theme designed for sites running Fluent Community. It stays out of the way on community portal pages (Fluent Community renders its own Vue.js SPA) while providing clean styling for regular WordPress pages.

Key Features

  • Zero interference with Fluent Community’s SPA rendering
  • Dark/light mode sync via CSS variables (bridges WordPress theme to Fluent Community colors)
  • Minimal chrome — blank-canvas approach with minimal header/footer
  • Block editor support for regular WordPress pages
  • Lightweight styling for blog posts and WooCommerce pages
This theme has no REST endpoints. It’s a visual WordPress theme only — the app doesn’t communicate with it directly. Install it on the site so non-community pages (blog, shop, etc.) look decent.

5. TBC OTP

companion plugins/tbc-otp/ Addon

Phone OTP verification via Twilio SMS or voice call. Hooks into tbc-community-app’s registration flow to add a phone verification step. Paired with the otp app module.

Endpoints

MethodEndpointAuthPurpose
POST/otp/sendNoValidate phone + start Twilio verification, return session key
POST/otp/verifyNoVerify the OTP code
POST/otp/resendNoResend OTP via SMS (60-second cooldown)
POST/otp/voiceNoSend code via voice call (Twilio fallback)

How It Works

  • Hooks tbc_ca_pre_register filter to intercept registration before user creation
  • Advertises OTP capability via tbc_ca_registration_config filter (app discovers it via /app-config)
  • Web registration: JS pre-submits OTP verification before form submission
  • App registration: Returns otp_required response — mobile app shows OTP step

Setup

  1. Install the plugin on WordPress
  2. In WP admin → TBC Community App → OTP Settings: enter your Twilio Account SID, Auth Token, and phone number or Messaging Service SID
  3. Enable the otp module in the app’s modules/_registry.ts
Completely optional. Without this plugin, registration works normally — the phone verification step simply won’t appear. The app adapts automatically via server config.

6. TBC Profile Completion

companion plugins/tbc-profile-completion/ Addon

Profile completion gate that requires users to fill in bio and/or avatar after registration before accessing the community. Works for both new registrations and returning users with incomplete profiles. Paired with the profile-completion app module.

How It Works

  • Hooks tbc_ca_post_register to mark new users as incomplete after registration
  • Returns X-TBC-Profile-Incomplete response header on API calls if profile is incomplete
  • Re-evaluates completion on every profile save (catches avatar/bio updates from any source)
  • Advertises capability via tbc_ca_registration_config filter
  • Web: redirects incomplete users to a full-page profile completion overlay on FC portal

Setup

  1. Install the plugin on WordPress
  2. In WP admin → TBC Community App → Profile Completion: configure which fields are required (bio, avatar)
  3. Enable the profile-completion module in the app’s modules/_registry.ts
Completely optional. Without this plugin, users go straight to the community after registration. The app adapts automatically via server config.

7. TBC YouTube

companion plugins/tbc-youtube/ Addon

YouTube channel integration. Fetches videos and playlists via the YouTube Data API v3, caches results server-side (6 hours), and serves them to the app. Paired with the youtube app module.

Endpoints

MethodEndpointAuthPurpose
GET/latestYesLatest uploads from channel (param: limit=1-15)
GET/playlistsYesAll channel playlists
GET/playlists/{id}/videosYesVideos in a playlist (params: limit, pageToken)
GET/configYesModule config (channel URL)

Setup

  1. Install the plugin on WordPress
  2. In WP admin → Settings → TBC YouTube: enter your YouTube Data API v3 key and channel ID
  3. Enable the youtube module in the app’s modules/_registry.ts

Caching

All YouTube API responses are cached via WordPress transients (6-hour TTL). The cache warms automatically when admin settings change. This keeps YouTube API quota usage low.

9. TBC License Server

companion plugins/tbc-license-server/ Infra — v2.2.0

Thin bridge between the buyer’s setup dashboard and FluentCart Pro’s licensing module. This plugin lives on the seller’s WordPress site (twobirdscode.com), not on buyer sites. It validates license keys, auto-activates them on first use, and returns update info in the format the dashboard expects.

Requires FluentCart Pro with the Licensing module enabled. FluentCart handles all license management (key generation, subscription sync, admin UI, customer portal, file hosting). This plugin only exposes the REST endpoint and handles auto-activation.

Endpoint

MethodEndpointAuthPurpose
POST/checkNoValidate license key, auto-activate on first use, return plan + expiry + latest version info

Request

POST /wp-json/tbc-license/v1/check
{
  "licenseKey": "TBC-XXXX-XXXX-XXXX",
  "currentVersion": "3.0.6",
  "product": "tbc-community-app",
  "siteUrl": "https://example.com"
}

The siteUrl field is the buyer’s site URL (from eas.json SITE_URL). It is sent with each validation request and used for auto-activation.

Response (valid, update available)

{
  "valid": true,
  "plan": "core",
  "expiresAt": "2027-03-16",
  "latest": {
    "version": "3.1.0",
    "date": "2026-03-20",
    "changelog": "### v3.1.0\n- Fluent Community 1.9 support",
    "downloadUrl": "https://twobirdscode.com/?fct-download=...",
    "size": 2048000
  }
}

Response (up to date)

{ "valid": true, "plan": "core", "expiresAt": "2027-03-16", "latest": null }

How It Works

  1. Looks up the license key in FluentCart’s fct_licenses table
  2. Checks expiration via FluentCart’s License model
  3. Auto-activation: if the license is inactive and has activations remaining, automatically activates it for the buyer’s site URL — creates records in fct_license_sites and fct_license_activations, then calls recountActivations() so the license shows as Active with the buyer’s site URL in FluentCart admin
  4. Checks active status (license must be active after auto-activation or already active)
  5. Reads version info from the product’s license_settings meta
  6. Generates a time-limited signed download URL via FluentCart’s Helper class (60 min validity)
  7. Returns the response in the dashboard’s expected format

MIME Type Support

The plugin registers tar.gz and gz MIME types (application/gzip) in WordPress so FluentCart update packages can be uploaded via the media library. Added in v2.1.0.

Not a buyer plugin. This is infrastructure for the TBC product — it runs on the seller’s server only. Buyers interact with it indirectly through their setup dashboard’s Updates tab. On first validation, their license is automatically activated — no manual activation step needed in FluentCart admin.

10. App-Side URL Constants

The app references plugin APIs via URL constants defined in constants/config.ts. All are derived from SITE_URL.

// Core plugin APIs
export const TBC_CA_URL  = `${SITE_URL}/wp-json/tbc-ca/v1`;       // Auth, registration, config
export const TBC_MR_URL  = `${SITE_URL}/wp-json/tbc-multi-reactions/v1`;

// Addon plugin APIs
export const TBC_OTP_URL = `${SITE_URL}/wp-json/tbc-otp/v1`;      // OTP (requires tbc-otp)
export const TBC_YT_URL  = `${SITE_URL}/wp-json/tbc-yt/v1`;       // YouTube (requires tbc-youtube)

// Also used:
export const API_URL     = `${SITE_URL}/wp-json/fluent-community/v2`;
export const WP_REST_URL = `${SITE_URL}/wp-json/wp/v2`;

Service files pass these as baseUrl to the HTTP client:

// services/api/registration.ts
const response = await fetch(`${TBC_CA_URL}/auth/register`, { ... });

// services/api/reactions.ts
const response = await get('/breakdown/feed/42', { baseUrl: TBC_MR_URL });

11. Installation

For Buyers (White-Label Setup)

  1. Upload the 2 core plugins + theme to wp-content/plugins/ and wp-content/themes/
  2. Activate all plugins and the theme in WordPress admin
  3. No configuration needed for tbc-community-app — it auto-detects Fluent Community
  4. Configure tbc-multi-reactions in WP admin to customize reaction icons (optional)

For Addon Plugins

  1. Upload the plugin to wp-content/plugins/ and activate
  2. Configure in WP admin (API keys, content, etc.)
  3. Enable the matching app module in modules/_registry.ts
  4. Rebuild the app

Plugin Updates

Plugins are updated manually (upload new version via FTP or WP admin). After updating a plugin:

  • Bump the plugin version number in the main PHP file header
  • Update the changelog in the plugin
  • Clear any relevant caches on the server
Always update the plugin version number — even for small changes. WordPress caches plugin metadata, and unchanged versions may not clear old cached responses.
