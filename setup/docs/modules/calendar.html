Overview

The Calendar module adds a full event calendar to the app — a dedicated tab, a monthly grid view, and two home screen widgets for upcoming bookings and featured events. Events are backed by WooCommerce products on the WordPress site, exposed via the companion plugin's REST API.

Setup

1. Install the companion plugin

Upload companion plugins/tbc-calendar-fluent to your WordPress site and activate it. The plugin requires WooCommerce to be installed and active.

2. Create events as WooCommerce products

Events are WooCommerce products. The companion plugin exposes them via REST API with event-specific fields (date, time, featured flag, etc.).

3. Enable the module

In modules/_registry.ts, ensure the calendar module is registered:

import { calendarModule } from './calendar/module';

export const modules = [
  calendarModule,
  // ...other modules
];

4. Add the route stub

Create (or verify) the tab route stub at app/(tabs)/calendar.tsx — a one-line re-export that makes the tab routable:

export { default } from '@/modules/calendar/screens/CalendarTab';

How It Works

The calendar tab displays a monthly grid. Tapping a date filters the event list below the grid to show events on that day. Events with a WooCommerce product link can be opened in a WebView for booking.

Widgets

Two home screen widgets surface event data without navigating to the calendar tab:

  • Ceremony widget — shows the user's next upcoming booking/ceremony
  • Featured Events widget — carousel of events flagged as featured

Both widgets link to the calendar tab via the "See All" action.

Both widgets are wrapped in the standard HomeWidget chrome (header bar with title, icon, and "See All" link) by the home screen.

hideMenuKey

The module sets hideMenuKey: 'calendar'. When the calendar tab is visible, any menu items with this key are hidden to avoid duplicate navigation paths.

API Endpoints

All endpoints use the base path /wp-json/tbc-wc/v1, provided by the tbc-calendar-fluent companion plugin.

WooCommerce required. The companion plugin depends on WooCommerce being installed and active on the WordPress site. Events are stored as WooCommerce products.

The API service at modules/calendar/services/calendarApi.ts wraps calls to these endpoints. Refer to the companion plugin source for the full endpoint specification.

WooCommerce Integration

Events in the calendar are WooCommerce products. This approach leverages WooCommerce's existing infrastructure for:

  • Product management — create/edit events in the WordPress admin using familiar WooCommerce UI
  • Bookings — users can book events directly via the WooCommerce checkout flow (opened in a WebView via the useEventWebView hook)
  • Payments — WooCommerce handles all payment processing

The companion plugin adds event-specific fields (date, time, featured flag) to WooCommerce products and exposes them through a dedicated REST namespace.
