1. Overview

Replaces the default single-heart like button with a multi-emoji reaction system. Users can tap to react with a default type, long-press to open a reaction picker, and tap the breakdown summary to see who reacted with what. Supports custom icon images (PNG, SVG, GIF, WEBP) configured in WordPress admin.

This module uses the slot system to inject reaction UI into core components (FeedCard, CommentItem) without modifying core code. When the module is removed from the registry, core components fall back to a simple like button.

2. Prerequisites

  • WordPress site with tbc-multi-reactions plugin installed and activated
  • Reaction types configured in WP admin → TBC Multi Reactions (defaults are provided)

3. Setup

  1. Install the tbc-multi-reactions companion plugin on your WordPress site
  2. Configure reaction types in WP admin → TBC Multi Reactions (emoji, color, icon image)
  3. The module is already registered in modules/_registry.ts — ships with the base product

To disable multi-reactions, comment out or remove multiReactionsModule from the MODULES array in modules/_registry.ts. Core components will fall back to a simple heart like button.

4. How It Works

Architecture

  • 4 slot components inject reaction UI into FeedCard and CommentItem
  • 1 context provider (MultiReactionsProvider) manages shared state for the reaction picker and breakdown modal
  • Core hook useReactionConfig fetches available reaction types from the plugin’s /config endpoint (stays in core since notifications and chat also use it)
  • Core hook useFeedReactions handles optimistic react/unreact/swap logic (stays in core since feed screens depend on it)

User interactions

ActionBehavior
Tap reaction buttonToggle default reaction type (or current type if already reacted)
Long-press reaction buttonOpen floating reaction picker
Select from pickerReact with chosen type (or swap if already reacted with different type)
Tap breakdown summaryOpen bottom sheet showing who reacted with what, filterable by reaction type

5. File Structure

modules/multi-reactions/
  module.ts                          ← Manifest (4 slots + 1 provider)
  MultiReactionsProvider.tsx         ← Context for picker & breakdown modal
  components/
    slotProps.ts                     ← Shared ReactionSlotProps interface
    FeedReactionSlot.tsx             ← Slot: feed reaction button (tap + long-press)
    FeedBreakdownSlot.tsx            ← Slot: feed emoji stack + count
    CommentReactionSlot.tsx          ← Slot: comment reaction button
    CommentBreakdownSlot.tsx         ← Slot: comment emoji stack + count
    ReactionBreakdownModal.tsx       ← Bottom sheet: who reacted with what

What stays in core (shared infrastructure)

  • components/feed/ReactionPicker.tsx — Floating picker UI (also used by chat)
  • components/feed/ReactionIcon.tsx — Pure icon renderer (also used by notifications)
  • hooks/useReactionConfig.ts — Fetches reaction config, graceful empty return if plugin not installed
  • hooks/useFeedReactions.ts — Optimistic react/swap logic for feed screens
  • services/api/feeds.ts — swapReactionType, getReactionBreakdown, getReactionBreakdownUsers (used by core hooks)

All slot components receive ReactionSlotProps:

interface ReactionSlotProps {
  objectId: number;
  hasReacted: boolean;
  userReactionType: string | null;
  userReactionIconUrl: string | null;
  reactionsCount: number;
  reactionBreakdown: ReactionBreakdown[];
  onReact: (type?: string) => void;
}

8. Fallback Behavior

When this module is not registered:

  • FeedCard renders a simple heart icon — tap to toggle like
  • CommentItem renders a simple heart icon — tap to toggle like
  • No picker, no breakdown modal, no emoji stack
  • Core reactToFeed still works (sends react_type: 'like' to Fluent Community)
  • The X-TBC-Reaction-Type header is harmless without the plugin installed
