Content Creation & Composer

Rich text editing, media attachments, documents, polls, space selection, and post submission

1. Composer Overview

The composer screen (/create-post) handles both creating new posts and editing existing ones. It's presented as a full-screen modal that slides up from the bottom.

3. Rich Text Editing (10tap)

The editor uses @10play/tentap-editor, a WebView-based rich text editor that outputs HTML.

Editor Setup

hooks/useThemedEditor.ts configures the editor bridge with theme-aware styling:

export function useThemedEditor({
  placeholder = 'Write a comment...',
  initialContent = '',
  autofocus = false,
}: UseThemedEditorOptions = {}) {
  const { colors } = useTheme();

  // Theme CSS applied via BridgeExtension.extendCSS — runs on WebView load,
  // no flash of unstyled content
  const themeCSS = useMemo(() => `
    body { color: ${colors.text}; background: ${colors.surface}; ... }
    a { color: ${colors.primary}; }
    blockquote { border-left: 3px solid ${colors.primary}; ... }
    code { background: ${colors.backgroundSecondary}; ... }
    /* ... more theme CSS ... */
  `, [colors]);

  const themeBridge = useMemo(
    () => new BridgeExtension({ forceName: 'tbcTheme', extendCSS: themeCSS }),
    [themeCSS],
  );

  const editor = useEditorBridge({
    initialContent,
    autofocus,
    avoidIosKeyboard: true,
    theme: { webview: { backgroundColor: colors.surface } },
    bridgeExtensions: [...TenTapStartKit, themeBridge],
  });

  // Set placeholder once editor WebView is ready (single-fire subscription)
  useEffect(() => {
    const unsub = editor._subscribeToEditorStateUpdate(() => {
      editor.setPlaceholder(placeholder);
      unsub();
    });
    return unsub;
  }, [editor, placeholder]);

  return editor;
}
Never use PlaceholderBridge.configureExtension(). It silently crashes the WebView. TenTapStartKit already includes PlaceholderBridge. Set the placeholder at runtime with editor.setPlaceholder(text) instead.

Dark Mode

The editor uses a custom BridgeExtension with extendCSS to inject theme CSS at WebView load time. This avoids flash of unstyled content. The theme.webview.backgroundColor on the editor bridge handles the WebView container background.

4. Markdown Toolbar

components/composer/MarkdownToolbar.tsx provides a horizontally scrollable formatting toolbar with group separators:

GroupButtonsFunction
InlineB, I, SBold, Italic, Strikethrough
HeadingsH2, H3, H4Heading levels (H1 reserved for title)
BlockBullet, Ordered, QuoteBullet list, Ordered list, Blockquote
OtherCode, LinkCode formatting, Link insertion

Active State

Each button reads the editor state via useBridgeState(editor) to highlight which formats are currently active. Active buttons get a primary-colored label/icon and a tinted background.

Link Insertion

Tapping the Link button replaces the toolbar row with an inline URL input field (not a modal). The user pastes a URL, taps the checkmark button to insert, or taps X to cancel. If a link is already active at the cursor, tapping Link removes it.

Compact Mode

The toolbar accepts a compact prop (used in comment sheets) that reduces button and font sizes.

5. HTML-to-Markdown Conversion

utils/htmlToMarkdown.ts converts the 10tap editor's HTML output to markdown for server submission:

  • Custom regex-based converter — no external dependencies (RN has no DOM, so turndown won't work)
  • Handles only 10tap's limited HTML subset (not arbitrary HTML)
  • Preserves code blocks by protecting them with placeholders before other conversions
  • Converts headings, lists, blockquotes, links, inline formatting
  • Strips <p> tags inside <li> elements (10tap wraps li content in paragraphs)
  • Decodes HTML entities (&amp;, &lt;, &gt;, &quot;, &#39;, &nbsp;)
  • Cleans up whitespace (collapses 3+ consecutive newlines to 2)
10tap's HTML subset only. This converter handles the limited tags 10tap produces (<h2>, <h3>, <h4>, <strong>, <em>, <del>, <ul>, <ol>, <blockquote>, <code>, <pre>, <a>, <hr>). It's not a general-purpose HTML-to-markdown converter.

6. Attachment Types

The composer supports five attachment types. Image, video, GIF, and poll buttons are always visible (active state is indicated by a highlighted icon). The document button is conditionally shown based on the selected space’s permissions, and documents are mutually exclusive with all other media (see Mutual Exclusion below).

Images

  • Picker: expo-image-picker with multi-selection (up to 4)
  • Upload: Each image uploaded via mediaApi.uploadMedia(uri, type, fileName, 'feed')
  • Preview: components/composer/MediaPreview.tsx shows horizontal thumbnail strip with remove buttons
  • Quality: 0.8 (80% JPEG compression)
  • Selection limit: 4 - attachments.length (accounts for already-attached images)

Video Embed

  • Input: components/composer/VideoAttachModal.tsx — BottomSheet with Oembed tab, URL input, and Embed button
  • Validation: Fetches oembed data via feedsApi.getOembed(url) (GET /feeds/oembed?url={url})
  • Preview: components/composer/VideoPreview.tsx shows thumbnail + title + provider
  • Supports: YouTube, Vimeo, Wistia, and other oembed providers
  • Behavior: Immediately attaches and closes the sheet after successful oembed fetch

GIF

  • Picker: components/composer/GifPickerModal.tsx — BottomSheet (75% height) with search input, 2-column grid, trending on open, debounced search (400ms), infinite scroll pagination
  • Preview: components/composer/GifPreview.tsx shows animated preview with remove button
  • API: giphyApi.searchGifs(query, offset) from services/api/giphy.ts — proxied through Fluent Community's GET /giphy endpoint
  • Attribution: "Powered by GIPHY" text at bottom of picker

Documents (FC Pro — Multipart Upload)

  • Picker: expo-document-picker with multi-select; native module so adding/removing this dep requires a new EAS dev client build (not OTA-safe)
  • Upload: POST /documents/upload (FC Pro) as multipart/form-data with space_id + file; uses the existing JWT, no companion-plugin proxy
  • Preview: components/composer/DocumentPreview.tsx renders vertical file rows; reuses iconForMime from utils/mime.ts (same util used by SpaceDocumentsSheet)
  • Permissions: The button only appears when space.permissions.can_upload_documents is true — FC’s FeedsHelper computes this from settings.document_library + settings.document_upload + the user’s role
  • Storage: Uploaded files become SpaceDocumentFile records ({ id, title, media_key, url, type, size }) attached to meta.document_lists on the post

Mutual Exclusion (when documents are attached)

Web FC Pro treats document posts as their own content_type. The mobile composer mirrors this:

  • While documents are attached, the image / GIF / poll / video buttons are hidden (onImagePress / etc. are passed undefined to ComposerToolbar)
  • The document button is hidden whenever any other media is attached
  • This matches the FC Pro web rule: a post is either a regular post (text + images / video / GIF / poll) or a document post — never both

ComposerToolbar

components/composer/ComposerToolbar.tsx renders the bottom action bar with attachment buttons on the left and a Submit button on the right:

ButtonActionActive / visibility
Image (icon)Opens image pickerShows spinner while uploading; hidden if documents attached
GIF (text badge)Opens GIF pickerPrimary color when GIF attached; hidden if documents attached
Poll (chart icon)Opens poll builderFilled icon + primary color when poll attached; hidden if documents attached

onImagePress, onGifPress, onPollPress, and onVideoPress are now optional on ComposerToolbar; passing undefined hides the corresponding button. CreatePostContent uses this to enforce mutual exclusion symmetrically — the toolbar itself has no awareness of attachment state.

7. Polls & Surveys

components/composer/PollBuilderSheet.tsx provides poll creation in a BottomSheet:

Poll Configuration

SettingOptions
Typesingle_choice or multi_choice (toggle switch labeled "Allow multiple answers")
Options2–4 text options (minimum 2 must be filled, max 100 chars each)
End dateNot yet implemented in UI (always sent as empty string)

Poll Preview

components/composer/PollPreview.tsx shows a card with option count and preview text. Tappable to edit, with a remove button.

Submission Format

// Survey data in post payload
survey: {
  type: 'single_choice',  // or 'multi_choice'
  options: [
    { label: 'Option 1', slug: 'opt_1' },
    { label: 'Option 2', slug: 'opt_2' },
  ],
  end_date: '',
}
Slugs are auto-generated. Option slugs follow the pattern opt_1, opt_2, etc. They're generated at submission time from the filtered (non-empty) options array. The server stores the poll configuration in meta.survey_config.

8. Space Selection

components/composer/SpaceSelector.tsx lets users choose which space to post in:

  • Trigger: Button showing selected space name or "Select a space..." placeholder
  • Modal: Full-screen pageSheet modal with searchable list grouped by space groups
  • Data: Fetches in parallel from spacesApi.getSpaceGroups({ options_only: true }) and spacesApi.getSpaces({ per_page: 100 })
  • Grouping: Spaces organized by parent_id matching group IDs, sorted by serial within groups; ungrouped spaces appear under "Other Spaces"
  • Search: Filters by space title or slug
  • Privacy icons: Lock (secret), people (private), globe (public)
  • Output: Returns space slug + title to parent component
Always send space slug, not ID. The Fluent Community API expects the space slug (e.g., 'general') in the space field, not the numeric ID. This matches the web app behavior.

The space selector is hidden in edit mode (you can't move a post to a different space) and when a spaceSlug param is already provided. When a space is pre-selected, a read-only indicator shows the space name instead.

9. Post Submission API

services/api/feeds.ts → createFeed(data)

ComposerSubmitData (what CreatePostContent builds)

interface ComposerSubmitData {
  message: string;           // Markdown content (converted from 10tap HTML)
  content_type: 'text' | 'markdown' | 'document';  // 'document' when meta.document_lists is present
  title?: string;
  space?: string;            // Space slug

  // Images (up to 4)
  media_images?: Array<{
    url: string;
    type: string;       // always 'image'
    width: number;
    height: number;
    provider: string;   // always 'uploader'
  }>;

  // Video embed (one)
  media?: { type: 'oembed'; url: string };

  // GIF
  meta?: {
    media_preview: {
      image: string;       // GIF URL
      type: 'image';
      provider: 'giphy';
      width: number;
      height: number;
    }
  };

  // Poll
  survey?: {
    type: 'single_choice' | 'multi_choice';
    options: { label: string; slug: string }[];
    end_date: string;       // Always '' (not yet implemented)
  };

  // Documents (FC Pro) — set content_type: 'document' when present
  meta?: {
    document_lists?: SpaceDocumentFile[];   // { id, title, media_key, url, type, size }
  };
}

createFeed() request (what goes to the server)

The createFeed() function in services/api/feeds.ts wraps the composer data with additional default fields to match the web app format:

POST /feeds

{
  type: 'text',                       // Default, set by createFeed()
  message: 'Markdown content here',   // From ComposerSubmitData
  space: 'general',                   // Space slug
  content_type: 'markdown',           // From ComposerSubmitData
  title: 'Optional Title',            // If provided

  // Always-included defaults (added by createFeed)
  media: null,                        // Overwritten if video attached
  media_image: '',
  topic_ids: [],
  send_announcement_email: 'no',

  // Conditionally included from ComposerSubmitData:
  media_images: [...],                // If images attached
  media: { type: 'oembed', url: '...' },  // If video attached (replaces null)
  meta: { media_preview: {...} },     // If GIF attached
  survey: {...},                      // If poll attached
}

Response

{
  success: true,
  data: { feed: Feed }  // The created feed object
}

After Submission

On success, the screen emits CACHE_EVENTS.FEEDS via cacheEvents.emit() to refresh the feed list, then the CreatePostContent component resets all state and calls onClose() which navigates back.

10. Post Editing

Edit mode is triggered by passing editId to the create-post screen. The existing post is fetched and all fields are populated.

Edit Flow

  1. Screen receives editId route param
  2. Fetches post via feedsApi.getFeedById(Number(editId))
  3. Populates editor with existing content:
    • Title and message body loaded (uses message_rendered falling back to message)
    • Existing images re-attached from meta.media_items (filtered to type 'image')
    • Video preview restored if meta.media_preview.provider is 'youtube'
    • GIF preview restored if meta.media_preview.provider is 'giphy'
    • Poll options extracted from meta.survey_config (when content_type is 'survey')
  4. Submit button label changes to "Save"
  5. Space selector is hidden (shows read-only space indicator instead)
  6. On save: calls feedsApi.updateFeed(id, data) (does not send space or media fields)
Same component, different mode. The composer doesn't have separate create/edit components. It's the same CreatePostContent with conditional behavior based on whether editFeed is present.

11. Keyboard Handling

The composer uses react-native-keyboard-controller with specific layout patterns:

  • KAV: KeyboardAvoidingView with behavior="padding"
  • Toolbar position: Absolutely positioned at bottom with bottom: insets.bottom
  • Media previews: Rendered inside the KAV, above the toolbar
  • Main content: Wrapped in a SafeAreaView with edges={['top']}
Toolbar uses absolute positioning exception. For the composer toolbar, safe area padding is applied directly on the KAV style as bottom: insets.bottom, NOT as a sibling spacer. This is the exception to the normal KAV pattern (see Architecture). The toolbar is absolutely positioned, so the sibling spacer pattern doesn't apply here.

13. Rules

  1. Always send content_type: 'markdown' for posts. The 10tap editor produces HTML, which htmlToMarkdown() converts before submission. The server expects markdown.
  2. Never use PlaceholderBridge.configureExtension(). It crashes the 10tap WebView. Use editor.setPlaceholder(text) at runtime instead. TenTapStartKit already includes PlaceholderBridge.
  3. Send space slug, not ID. The space field in the API payload is the space's URL slug (e.g., 'general'), not its numeric database ID.
  4. Images go in media_images, videos in media, GIFs in meta.media_preview. These are three different fields with different structures. Don't mix them up.
  5. Poll option slugs are auto-generated. Use the opt_1, opt_2 pattern. Don't let users set custom slugs. The server uses these for vote tracking.
  6. Max 4 images per post. The picker's selectionLimit accounts for already-attached images. This limit matches the Fluent Community web UI.
  7. Use useThemedEditor() for all 10tap instances. The hook handles theme CSS via BridgeExtension.extendCSS for flash-free theming. It's shared by the composer, comment sheet, and any module comment screens.
  8. Emit CACHE_EVENTS.FEEDS after successful create/update. This triggers feed list refresh so the new or edited post appears immediately. Don't navigate back without emitting.
  9. htmlToMarkdown() handles only 10tap's HTML subset. It's not a general-purpose converter. Don't use it for arbitrary HTML from the server or external sources.
  10. Server enforces posting permissions. The composer doesn't pre-check whether the user can post in a space. If the user lacks permission, the API returns an error and the composer shows an alert. Don't add client-side permission checks that could get out of sync with the server.
  11. GIF provider is 'giphy' on submit. The composer sends provider: 'giphy' in meta.media_preview, matching what the native FluentCommunity web app sends. The server's processFeedMetaData requires provider == 'giphy' to persist the GIF — anything else is silently dropped (and the updateFeed path has no fallback shim, so edits would lose the GIF).
  12. Documents are mutually exclusive with all other media. When meta.document_lists has items, set content_type: 'document' and pass undefined for onImagePress / onGifPress / onPollPress / onVideoPress on ComposerToolbar. The document button is hidden whenever any other media is attached. This mirrors web FC Pro — a post is either a regular post or a document post, never both.
  13. Gate the document button on space.permissions.can_upload_documents. FC’s FeedsHelper already computes this from settings.document_library + settings.document_upload + the user’s role. Don’t re-derive it client-side; just read the boolean.
