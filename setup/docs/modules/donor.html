Overview

The Donor Dashboard module is the simplest possible module — a single launcher item that opens a WordPress page in the in-app WebView browser. No tab, no widget, no screen component. Just one file.

Setup

1. Create the WordPress page

Create a page on your WordPress site at the path you want to use (e.g. /donor-dashboard/). This page should contain whatever donor-facing content you want — giving history, receipts, recurring donations, etc.

2. Configure the URL

In modules/donor/module.ts, update the URL in the route params:

route: {
  pathname: '/webview',
  params: {
    url: `${SITE_URL}/donor-dashboard/`,  // SETUP: Path to your donor dashboard page
    title: 'Donor Dashboard',
  },
},

3. Enable the module

In modules/_registry.ts, ensure the donor module is registered:

import { donorModule } from './donor/module';

export const modules = [
  donorModule,
  // ...other modules
];
No route stub needed. Since this module only registers a launcher item (no tab), there is no route stub file in app/(tabs)/.

How It Works

When the user opens the Launcher and taps "Donor Dashboard":

  1. The launcher item's route navigates to /webview
  2. The WebView screen loads the configured WordPress page URL
  3. The user interacts with the donor dashboard inside the in-app browser

The WebView inherits the user's authentication session, so they see their own donor data without needing to log in again.

Module Manifest

The complete manifest in modules/donor/module.ts:

import type { ModuleManifest } from '@/modules/_types';
import { SITE_URL } from '@/constants/config';

export const donorModule: ModuleManifest = {
  id: 'donor',
  name: 'Donor Dashboard',
  version: '1.0.0',
  description: 'Donor Dashboard launcher item (WebView)',
  author: 'Two Birds Code',
  authorUrl: 'https://twobirdscode.com',
  license: 'Proprietary',

  launcherItems: [
    {
      id: 'donor-dashboard',
      label: 'Donor Dashboard',
      icon: 'wallet-outline',
      route: {
        pathname: '/webview',
        params: {
          url: `${SITE_URL}/donor-dashboard/`,
          title: 'Donor Dashboard',
        },
      },
      order: 65,
      hideKey: 'donor_dashboard',
    },
  ],
};

The "Launcher-Only" Pattern

This module demonstrates the simplest possible module architecture — a single manifest file with one launcher item. It is a useful template for any feature that just needs to open an existing WordPress page in the app.

Common use cases for this pattern:

  • Donor/giving dashboards
  • Account settings pages
  • External forms or surveys
  • Support/help pages
  • Any WordPress page you want accessible from the app menu
Minimal footprint. A launcher-only module requires:
  • One file: modules/yourmodule/module.ts
  • One import + one array entry in modules/_registry.ts
  • No route stubs, no components, no services
hideKey controls visibility. If you also add a tab or widget for the same feature, use a matching hideKey so the launcher item hides when the tab is visible (preventing duplicate navigation).
