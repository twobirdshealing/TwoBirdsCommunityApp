1. Upload Pipeline Overview

The complete pipeline from user action to display:

User taps image icon
  → ImagePicker.launchImageLibraryAsync() [quality 0.8, max 4]
  → For each asset:
      uploadMedia(uri, mimeType, fileName, 'feed')
        → FormData { file, object_source }
        → POST /feeds/media-upload (rawBody: true)
        → Returns { media: { url, width, height, type, media_id } }
      Add to attachments[] with dimensions
  → Show MediaPreview thumbnails
  → User submits post
  → Serialize as media_images: [{ url, type, width, height, provider: 'uploader' }]
  → POST /feeds with media_images
  → Feed renders responsive image grid
  → Tap image → MediaViewer (pinch zoom, save, share)

2. Image & Video Picking

Avatar Picker Utility

utils/avatarPicker.ts provides platform-aware image picking:

import { showAvatarPicker, showCoverPicker, showImagePicker } from '@/utils/avatarPicker';

// Avatar (1:1 aspect ratio)
showAvatarPicker({
  onImageSelected: (uri, type, fileName) => { /* upload */ },
  onError: (msg) => Alert.alert('Error', msg),
});

// Cover photo (16:9 aspect ratio)
showCoverPicker({
  onImageSelected: (uri, type, fileName) => { /* upload */ },
  onError: (msg) => Alert.alert('Error', msg),
});
  • iOS: Shows ActionSheetIOS with Camera / Photo Library options
  • Android: Shows Alert with the same options
  • Permissions: Requests camera or media library permission before launching
  • Quality: 0.8 (80% JPEG compression) by default
  • Editing: allowsEditing: true — lets the user crop before confirming

Multi-Image Picking (Post Composer)

In components/composer/CreatePostContent.tsx, the composer uses expo-image-picker directly for multi-selection:

const result = await ImagePicker.launchImageLibraryAsync({
  mediaTypes: ['images'],
  allowsMultipleSelection: true,
  selectionLimit: 4 - attachments.length,  // Max 4 total
  quality: 0.8,
});
Max 4 images per post. The selection limit is dynamic — if 2 images are already attached, the picker allows selecting 2 more. This matches the Fluent Community web limit.

3. Upload API

services/api/media.ts contains the single upload function:

export async function uploadMedia(
  uri: string,
  type: string,         // e.g. 'image/jpeg'
  fileName: string,
  objectSource: string  // 'feed' | 'profile' | 'comment'
): Promise<MediaUploadResponse>

How It Works

  1. Creates a FormData with the file in React Native format: { uri, type, name }
  2. Appends object_source to identify the upload context
  3. Sends POST /feeds/media-upload with rawBody: true
  4. The rawBody flag tells the HTTP client to send FormData as-is (not JSON-stringified) and let FormData set its own Content-Type with boundary

Response Type

interface MediaUploadResponse {
  success: boolean;
  data?: {
    media_id: number;
    url: string;
    type: string;
    width?: number;
    height?: number;
    mime_type?: string;
  };
  error?: { code: string; message: string };
}
Never set Content-Type manually for uploads. The HTTP client skips the Content-Type header when rawBody is true. FormData must set its own boundary. Manually setting 'multipart/form-data' breaks the upload.

4. Media in Feed Posts

components/feed/FeedCard.tsx detects media type from the feed object using a priority chain:

PriorityCheckMedia Type
1meta.media_items array existsMulti-image or video items
2meta.media_preview?.provider === 'youtube'YouTube embed
3meta.media_preview?.content_type === 'video'Direct video file
4meta.media_preview?.imageSingle image (from preview)
5feed.featured_imageLegacy featured image
6YouTube link in message textYouTube fallback

Submission Format

When creating a post with images:

// Images
submitData.media_images = attachments.map(item => ({
  url: item.url,
  type: item.type,
  width: item.width,
  height: item.height,
  provider: 'uploader',
}));

// Video embed (YouTube, Vimeo, etc.)
submitData.media = { type: 'oembed', url: videoUrl };

// GIF
submitData.meta = {
  media_preview: {
    image: gifUrl,
    type: 'image',
    provider: 'inline',  // Server converts to 'giphy'
    width: gifWidth,
    height: gifHeight,
  },
};

6. Gallery Viewer (Lightbox)

components/media/MediaViewer.tsx provides a fullscreen image viewer:

Features

  • Horizontal paging — FlatList with pagingEnabled for swipe between images
  • Pinch-to-zoom — react-native-reanimated gesture handler
  • Double-tap to zoom — Toggles between 1x and 2.5x scale
  • Pan when zoomed — Two-finger pan gesture for zoomed images
  • Image counter — Shows "3/10" when multiple images
  • Thumbnail strip — 48x48 thumbnail row at bottom for quick navigation
  • Auto zoom reset — Resets to 1x when swiping to next image

Top Bar Actions

ActionImplementation
CloseTop-left X button, dismisses modal
Save to GalleryDownloads via expo-file-system, saves via expo-media-library
ShareDownloads to cache, opens native share sheet via expo-sharing

Usage

<MediaViewer
  visible={showViewer}
  images={imageUrls}         // string[] of image URLs
  initialIndex={tappedIndex} // Start at the image user tapped
  onClose={() => setShowViewer(false)}
/>

7. Video Playback

components/media/VideoPlayer.tsx handles direct video files using expo-video:

const player = useVideoPlayer(url, (p) => {
  p.loop = false;
});
  • Shows poster image until first frame renders
  • Native video controls enabled
  • Fixed 16:9 aspect ratio
  • Loading indicator overlay while buffering
  • Callbacks: onPlay, onEnd

Video in Feed Posts

Videos are lazy-loaded in FeedCard. Initially shows a black background with a play button overlay. Tapping mounts the VideoPlayer component. This avoids loading heavy video components for every post in the feed.

9. GIF Picker & Display

GIF Picker Modal

components/composer/GifPickerModal.tsx provides a searchable GIF browser powered by the Giphy API:

  • Data source: Server-proxied Giphy API via services/api/giphy.ts
  • Search: Text input with 400ms debounce; empty query shows trending
  • Layout: 2-column masonry grid
  • Pagination: Infinite scroll, 20 GIFs per page
  • Selection: Tap selects a GIF, uses downsized_medium URL for submission and preview_gif for thumbnails

GIF Types

// types/gif.ts
interface GifAttachment {
  image: string;       // downsized_medium URL (for submission)
  width: number;
  height: number;
  previewUrl: string;  // preview_gif URL (for display)
}

GIF API

// services/api/giphy.ts
searchGifs(query?: string, offset?: number)
// Empty query → trending GIFs
// Returns: { gifs: GiphyGif[] }
GIF provider naming. GIFs are submitted with provider: 'inline' but the server converts this to 'giphy' on save. When reading feed data, check for provider === 'giphy', not 'inline'.

10. Avatar & Cover Photos

ProfilePhotoPicker Component

components/common/ProfilePhotoPicker.tsx displays the cover photo with overlapping avatar, both tappable:

  • Cover photo: full-width banner with camera icon overlay
  • Avatar: circular image overlapping the cover bottom edge, with camera icon
  • Both show loading spinners during upload
  • Uses expo-image with cachePolicy="memory-disk"

Upload Flow

  1. User taps avatar/cover → showAvatarPicker() or showCoverPicker()
  2. Camera or library picker opens with appropriate aspect ratio (1:1 or 16:9)
  3. User crops and confirms
  4. uploadMedia(uri, type, fileName, 'profile') sends to server
  5. On success → callback returns remote URL
  6. Caller stores URL with PUT /profile/{id}

Avatar Display Component

components/common/Avatar.tsx renders user avatars throughout the app with a text fallback (first letter of name) when no image is available.

Video Embed Modal

  1. User enters URL in text input
  2. App fetches oembed data via GET /feeds/oembed?url={url}

13. Rules

  1. All uploads go through uploadMedia(). Never construct FormData or call the upload endpoint directly. The function handles the correct React Native file format, rawBody flag, and error response parsing.
  2. Never set Content-Type for uploads. The HTTP client skips Content-Type when rawBody: true. FormData sets its own multipart boundary. Manually setting the header breaks the upload.
  3. Use expo-image with cachePolicy="memory-disk" for display. All image rendering uses this for consistent caching. Don't use React Native's built-in Image component for remote images.
  4. Max 4 images per post. The picker's selectionLimit is dynamic: 4 - attachments.length. Don't increase this limit — it matches the Fluent Community web UI.
  5. Use showAvatarPicker() / showCoverPicker() for profile images. These handle platform-specific UI (ActionSheetIOS vs Alert), permissions, and aspect ratio locking. Don't call ImagePicker directly for avatar/cover.
  6. Videos are lazy-loaded in feed. FeedCard shows a thumbnail with play overlay. The actual VideoPlayer or YouTubeEmbed mounts only on tap. Don't pre-mount video components in list items.
  7. GIFs use downsized_medium for submission, preview_gif for display. The downsized version is what gets stored on the server. The preview version is used in the picker grid and composer preview.
  8. YouTube embed is an optional dependency. react-native-youtube-iframe has a graceful fallback to browser. Don't make it a hard requirement. Check for its presence before importing.
  9. Pass initialIndex to MediaViewer. When opening the gallery from a grid, pass the index of the tapped image so the viewer starts at the correct position.
  10. Always pass object_source to uploadMedia(). Use 'feed' for post images, 'profile' for avatars/covers, 'comment' for comment media. The server uses this for organization and cleanup.
