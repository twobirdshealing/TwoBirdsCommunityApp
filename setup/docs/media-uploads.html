1. Upload Pipeline Overview

The complete pipeline from user action to display:

User taps image icon
  → ImagePicker.launchImageLibraryAsync() [quality 0.8, max 4]
  → For each asset:
      uploadMedia(uri, mimeType, fileName, 'feed')
        → FormData { file, object_source }
        → POST /feeds/media-upload (rawBody: true)
        → Returns { media: { url, width, height, type, media_key } }
      Add to attachments[] with dimensions
  → Show MediaPreview thumbnails
  → User submits post
  → Serialize as media_images: [{ url, type, width, height, provider: 'uploader' }]
  → POST /feeds with media_images
  → Feed renders responsive image grid
  → Tap image → MediaViewer (pinch zoom, save, share)

2. Image & Video Picking

Image Picker Utility

utils/avatarPicker.ts provides platform-aware image picking with built-in upload:

import { showAvatarPicker, showCoverPicker, showImagePicker } from '@/utils/avatarPicker';

// Avatar (1:1 aspect ratio)
showAvatarPicker({
  onUploadStart: (localUri) => { /* show spinner */ },
  onSuccess: (remoteUrl) => { /* save URL */ },
  onError: (msg) => Alert.alert('Error', msg),
});

// Cover photo (16:9 aspect ratio)
showCoverPicker({
  onUploadStart: (localUri) => { /* show spinner */ },
  onSuccess: (remoteUrl) => { /* save URL */ },
  onError: (msg) => Alert.alert('Error', msg),
});

// Generic picker with custom options
showImagePicker(callbacks, {
  aspect: [4, 3],
  quality: 0.9,
  defaultFileName: 'photo.jpg',
  objectSource: 'profile',
});

Callbacks Interface

interface ImagePickerCallbacks {
  onUploadStart?: (localUri: string) => void;
  onSuccess: (remoteUrl: string) => void;
  onError: (message: string) => void;
}
  • iOS: Shows ActionSheetIOS with Take Photo / Choose from Library options
  • Android: Shows Alert with the same options
  • Permissions: Requests camera permission before launching camera (library picker in avatarPicker skips explicit permission; the multi-image composer picker requests MediaLibraryPermissions before launch)
  • Quality: 0.8 (80% JPEG compression) by default
  • Editing: allowsEditing: true — lets the user crop before confirming
  • Upload: The picker handles upload internally via uploadMedia() — onSuccess receives the remote URL directly

Multi-Image Picking (Post Composer)

In components/composer/CreatePostContent.tsx, the composer uses expo-image-picker directly for multi-selection:

const result = await ImagePicker.launchImageLibraryAsync({
  mediaTypes: ['images'],
  allowsMultipleSelection: true,
  selectionLimit: 4 - attachments.length,  // Max 4 total
  quality: 0.8,
});
Max 4 images per post. The selection limit is dynamic — if 2 images are already attached, the picker allows selecting 2 more. This matches the Fluent Community web limit.

3. Upload API

services/api/media.ts contains the single upload function:

export async function uploadMedia(
  uri: string,
  type: string,           // e.g. 'image/jpeg'
  fileName: string,
  objectSource: string = 'feed'  // 'feed' | 'profile' | 'comment'
): Promise<MediaUploadResponse>

How It Works

  1. Creates a FormData with the file in React Native format: { uri, type, name }
  2. Appends object_source to identify the upload context
  3. Sends POST /feeds/media-upload with rawBody: true
  4. The rawBody flag tells the HTTP client to send FormData as-is (not JSON-stringified) and let FormData set its own Content-Type with boundary
  5. Normalizes the response: extracts media_id from media_key, media_id, or id (whichever the API returns)

Response Type

interface MediaUploadResponse {
  success: boolean;
  data?: {
    media_id: number;
    url: string;
    type: string;
    width?: number;
    height?: number;
    mime_type?: string;
  };
  error?: { code: string; message: string };
}
Never set Content-Type manually for uploads. The HTTP client skips the Content-Type header when rawBody is true. FormData must set its own boundary. Manually setting 'multipart/form-data' breaks the upload.

4. Media in Feed Posts

components/feed/FeedCard.tsx detects media type from the feed object using a priority chain:

PriorityCheckMedia Type
1meta.media_items array existsMulti-image or video items
2meta.media_preview?.provider === 'youtube' + content_type === 'video' + urlYouTube embed
3meta.media_preview?.content_type === 'video' + urlDirect video file
4meta.media_preview?.image (skip if provider === 'youtube')Single image (from preview)
5feed.featured_imageLegacy featured image
6YouTube link in message textYouTube fallback

Submission Format

When creating a post with images:

// Images
submitData.media_images = attachments.map(item => ({
  url: item.url,
  type: 'image',
  width: 0,
  height: 0,
  provider: 'uploader',
}));

// Video embed (YouTube, Vimeo, etc.)
submitData.media = { type: 'oembed', url: videoUrl };

// GIF
submitData.meta = {
  media_preview: {
    image: gifUrl,
    type: 'image',
    provider: 'inline',  // Server converts to 'giphy'
    width: gifWidth,
    height: gifHeight,
  },
};

6. Gallery Viewer (Lightbox)

components/media/MediaViewer.tsx provides a fullscreen image viewer:

Features

  • Horizontal paging — horizontal ScrollView with pagingEnabled; index tracked via onMomentumScrollEnd
  • Pinch-to-zoom — react-native-reanimated + react-native-gesture-handler
  • Double-tap to zoom — Toggles between 1x and 2.5x scale
  • Pan when zoomed — Two-finger pan gesture (minPointers(2)) for zoomed images
  • Image counter — Shows "1 / 3" pill when multiple images
  • Thumbnail strip — 48x48 thumbnail row at bottom for quick navigation (active thumbnail highlighted with white border)
  • Auto zoom reset — Resets to 1x when zoomed below 1x (snaps back)

Top Bar Actions

ActionImplementation
CloseTop-left X button, dismisses modal
Save to GalleryDownloads via File.downloadFileAsync() (expo-file-system SDK 54), saves via expo-media-library
ShareDownloads to cache via File.downloadFileAsync(), opens native share sheet via expo-sharing

Usage

interface MediaViewerImage {
  url: string;
  width?: number;
  height?: number;
}

<MediaViewer
  visible={showViewer}
  images={imageArray}          // MediaViewerImage[] (objects with url, width?, height?)
  initialIndex={tappedIndex}   // Start at the image user tapped
  onClose={() => setShowViewer(false)}
/>

7. Video Playback

components/media/VideoPlayer.tsx handles direct video files using expo-video:

const player = useVideoPlayer(url, (p) => {
  p.loop = false;
});
  • Shows poster image until first frame renders (via onFirstFrameRender)
  • Native video controls enabled (nativeControls on VideoView)
  • Fixed 16:9 aspect ratio
  • Loading indicator overlay while status is 'loading' or 'idle'
  • Callbacks: onPlay (fires on playingChange), onEnd (fires on playToEnd, resets to start)
  • Uses useEvent and useEventListener from expo for player events

Video in Feed Posts

Videos are lazy-loaded in FeedCard. Initially shows a black background with a play button overlay. Tapping mounts the VideoPlayer component. This avoids loading heavy video components for every post in the feed.

8. OEmbed Player

components/media/OEmbedPlayer.tsx renders non-YouTube video embeds (Vimeo, Wistia, etc.) using a WebView:

  • Takes raw oEmbed HTML (typically an <iframe>) from the API
  • Wraps the HTML in a responsive 16:9 container with CSS
  • Uses react-native-webview with inline media playback enabled
  • Shows loading indicator until WebView finishes loading
  • Scroll disabled, JavaScript enabled

Usage

interface OEmbedPlayerProps {
  html: string;  // Raw oEmbed HTML from the API
}

<OEmbedPlayer html={oembedHtml} />

9. YouTube Embeds

components/media/YouTubeEmbed.tsx uses react-native-youtube-iframe for embedded playback:

  • Video ID extracted via extractYouTubeId(url) from utils/youtube.ts
  • Supports URL formats: youtube.com/watch?v=, youtu.be/, youtube.com/embed/
  • Loading overlay with spinner while iframe loads
  • Parent controls play state via playing prop
  • Player width adapts to container via onLayout, height calculated as 9/16 ratio

Play Button Overlay

components/media/PlayButtonOverlay.tsx renders on top of video thumbnails:

VariantStyle
'youtube'Red 48×34px pill with play icon, plus YouTube logo badge (top-left)
'video'White 48×48px circle with play icon
Optional dependency: react-native-youtube-iframe is loaded via a try/catch require(). If the package is not installed, tapping the YouTube thumbnail opens the video in the browser via Linking.openURL(). The fallback shows a red "Open in YouTube" button.

10. GIF Picker & Display

GIF Picker Modal

components/composer/GifPickerModal.tsx provides a searchable GIF browser powered by the Giphy API:

  • Data source: Server-proxied Giphy API via services/api/giphy.ts — GET /giphy?q={search}&offset={offset}
  • Search: Text input with 400ms debounce; empty query shows trending
  • Layout: 2-column grid using BottomSheetFlatList with numColumns={2}
  • Pagination: Infinite scroll, 20 GIFs per page
  • Selection: Tap selects a GIF, uses downsized_medium URL for submission and preview_gif for thumbnails
  • Attribution: "Powered by GIPHY" footer text
  • Container: BottomSheet at 75% height with search input using SheetInput wrapper

GIF Types

// types/gif.ts
interface GifAttachment {
  image: string;       // downsized_medium URL (used for post submission)
  width: number;
  height: number;
  previewUrl: string;  // preview_gif URL (used for thumbnail display)
}

GIF API

// services/api/giphy.ts
searchGifs(query: string = '', offset: number = 0)
// Empty query → trending GIFs
// Calls: GET /giphy?q={query}&offset={offset}
// Returns: { gifs: GiphyGif[] }

interface GiphyGif {
  images: {
    preview_gif: GiphyImage;      // { url, width, height }
    downsized_medium: GiphyImage;  // { url, width, height }
  };
}
GIF provider naming. GIFs are submitted with provider: 'inline' but the server converts this to 'giphy' on save. When reading feed data, check for provider === 'giphy', not 'inline'.

11. Avatar & Cover Photos

ProfilePhotoPicker Component

components/common/ProfilePhotoPicker.tsx displays the cover photo with overlapping avatar, both tappable:

  • Cover photo: full-width banner with camera icon overlay
  • Avatar: circular image overlapping the cover bottom edge, with camera icon
  • Both show loading spinners during upload
  • Uses expo-image with cachePolicy="memory-disk"

Upload Flow

  1. User taps avatar/cover → showAvatarPicker(callbacks) or showCoverPicker(callbacks)
  2. Camera or library picker opens with appropriate aspect ratio (1:1 or 16:9)
  3. User crops and confirms
  4. onUploadStart(localUri) fires — caller shows spinner
  5. Picker internally calls uploadMedia(uri, type, fileName, 'profile')
  6. On success → onSuccess(remoteUrl) fires with the remote URL
  7. Caller stores URL with PUT /profile/{id}

Avatar Display Component

components/common/Avatar.tsx renders user avatars throughout the app with a text fallback (first letter of name) when no image is available.

12. Link Previews

components/media/LinkPreview.tsx renders cards for non-video oembed content:

  • Two modes: compact (no thumbnail — row layout with icon, title, domain) and full (with thumbnail image, provider badge, title, description)
  • Provider detection from domain (Instagram, TikTok, Twitter/X, Reddit, Facebook, LinkedIn, Spotify, Pinterest, GitHub, Medium)
  • Provider icons shown as emoji identifiers
  • Tapping opens URL in browser via Linking.openURL()

Video Embed Modal

components/composer/VideoAttachModal.tsx lets users paste a URL to embed:

  1. Opens as a BottomSheet with "Oembed" and "Custom HTML Code" tabs (oembed only functional)
  2. User enters URL in text input
  3. App fetches oembed data via feedsApi.getOembed(url) which calls GET /feeds/oembed?url={url}
  4. On success, immediately attaches the video and closes the sheet
  5. On submit, video attachment is included as media: { type: 'oembed', url }

14. Rules

  1. All uploads go through uploadMedia(). Never construct FormData or call the upload endpoint directly. The function handles the correct React Native file format, rawBody flag, and error response parsing.
  2. Never set Content-Type for uploads. The HTTP client skips Content-Type when rawBody: true. FormData sets its own multipart boundary. Manually setting the header breaks the upload.
  3. Use expo-image with cachePolicy="memory-disk" for display. All image rendering uses this for consistent caching. Don't use React Native's built-in Image component for remote images.
  4. Max 4 images per post. The picker's selectionLimit is dynamic: 4 - attachments.length. Don't increase this limit — it matches the Fluent Community web UI.
  5. Use showAvatarPicker() / showCoverPicker() for profile images. These handle platform-specific UI (ActionSheetIOS vs Alert), permissions, aspect ratio locking, and upload. Don't call ImagePicker directly for avatar/cover.
  6. Videos are lazy-loaded in feed. FeedCard shows a thumbnail with play overlay. The actual VideoPlayer or YouTubeEmbed mounts only on tap. Don't pre-mount video components in list items.
  7. GIFs use downsized_medium for submission, preview_gif for display. The downsized version is what gets stored on the server. The preview version is used in the picker grid and composer preview.
  8. YouTube embed is an optional dependency. react-native-youtube-iframe is loaded via try/catch require() with a graceful fallback to browser. Don't make it a hard requirement.
  9. Pass initialIndex to MediaViewer. When opening the gallery from a grid, pass the index of the tapped image so the viewer starts at the correct position.
  10. Always pass object_source to uploadMedia(). Use 'feed' for post images, 'profile' for avatars/covers, 'comment' for comment media. The server uses this for organization and cleanup.
  11. Use OEmbedPlayer for non-YouTube video embeds. Vimeo, Wistia, and other providers render via WebView using the raw oEmbed HTML from the API. Don't try to parse their URLs into native players.
