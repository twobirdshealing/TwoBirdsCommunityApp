1. Overview

Courses are structured learning content managed through Fluent Community's LMS. The app provides:

  • Course browsing — all courses and enrolled courses tabs with search
  • Course detail — sections with collapsible lesson lists, progress tracking
  • Lesson viewer — HTML content, YouTube videos, documents, comments
  • Progress — per-lesson completion with optimistic updates
  • Home widget — carousel of enrolled courses on the home screen

Navigation Flow

Courses List (/courses)
  → Course Detail (/courses/[slug])
    → Lesson View (/courses/[slug]/lesson/[lessonSlug])
      → Previous / Next lesson navigation

2. Data Types

types/course.ts defines all course-related interfaces:

Course

interface Course {
  id: number;
  title: string;
  slug: string;
  description: string;
  cover_photo: string | null;
  status: 'published' | 'draft';
  privacy: 'public' | 'private' | 'secret';
  settings: CourseSettings;
  instructor: CourseInstructor;
  sections_count: number;
  lessons_count: number;
  students_count: number;
  sections?: CourseSection[];      // Detail endpoint only
  track?: CourseTrack;             // User's progress
  lockscreen_config?: CourseLockscreenConfig;
}

FieldDescription

CourseSection & CourseLesson

interface CourseSection {
  id: number;
  title: string;
  slug: string;
  serial: number;
  lessons: CourseLesson[];
  is_locked: boolean;
  unlock_date?: string;
}

interface CourseLesson {
  id: number;
  title: string;
  slug: string;
  serial: number;
  content: string;               // HTML (may be empty if lazy_load)
  content_type: string;          // 'text', 'quiz', etc.
  featured_image: string | null;
  is_locked: boolean;
  is_viewable: boolean;
  meta: CourseLessonMeta;
}

CourseTrack (Progress)

interface CourseTrack {
  completed_lessons: string[];   // Array of lesson IDs (as strings)
  isEnrolled: boolean;
  progress: number;              // 0-100 percentage
}

3. Courses API

services/api/courses.ts provides all course operations:

FunctionEndpointDescription
getCourses(options)GET /coursesList courses (paginated, searchable, filterable)
getCourseBySlug(slug, intendedLessonSlug?)GET /courses/{slug}/by-slugCourse detail with sections and lessons
getLessonBySlug(courseSlug, lessonSlug)GET /courses/{slug}/lessons/{lessonSlug}/by-slugIndividual lesson with full content
enrollInCourse(courseId)POST /courses/{id}/enrollEnroll current user
toggleLessonCompletion(courseId, lessonId, state)PUT /courses/{id}/lessons/{lessonId}/completionMark lesson complete or incomplete

Query Options

interface GetCoursesOptions {
  page?: number;
  per_page?: number;
  search?: string;
  topic_slug?: string;        // Filter by category
  type?: 'enrolled';          // Only enrolled courses
  sort_by?: 'alphabetical' | 'oldest' | 'latest';
  with_categories?: boolean;  // Include category list
}
Intended lesson slug optimization. When navigating to a specific lesson via deep link, pass intendedLessonSlug to getCourseBySlug(). The server pre-populates that lesson's content, avoiding a second API call.

4. Course List Screen

app/courses/index.tsx — route: /courses

  • Tabs: "All Courses" and "My Courses" (enrolled only)
  • Search: Text input with 400ms debounce
  • List: FlashList with infinite scroll pagination
  • Pull-to-refresh: Resets pagination and refetches
  • Cards: CourseCard with cover photo, title, stats, and progress bar for enrolled courses

5. Course Detail Screen

app/courses/[slug].tsx — route: /courses/[slug]

Layout

  1. Hero section — cover photo (or placeholder icon) with emoji badge and stats overlay (sections, lessons, students)
  2. Instructor card — avatar + name + student count (tappable to profile, hidden if hide_instructor_view)
  3. Enrollment CTA — "Enroll in Course" button, or progress bar + "Continue Learning" if enrolled
  4. Course details — rich text description section
  5. Sections list — collapsible sections with lesson rows showing lock/check/number indicators

Continue Learning

The "Continue Learning" button navigates to the first incomplete lesson. If all lessons are complete, it navigates to the first lesson in the course.

6. Lesson View Screen

app/courses/[slug]/lesson/[lessonSlug].tsx — route: /courses/[slug]/lesson/[lessonSlug]

Layout

  1. Hero image — extracted from the first <img> or <figure> in the lesson HTML (removed from body to avoid duplication)
  2. YouTube video — if lesson.meta.media.url contains a YouTube URL
  3. Lesson body — remaining HTML rendered via HtmlContent component
  4. Documents — downloadable files from lesson.meta.document_lists
  5. Comments — if enabled, shows comment count badge and links to comments screen
  6. Sticky bottom bar — state-specific UI (see below)

Bottom Bar States

StateUI
Not enrolledPrevious / Next navigation buttons
Enrolled, not complete"Mark as Complete" button
CompletingLoading spinner + success animation
CompletedSuccess badge + Previous / Next navigation

Lesson Navigation

Previous/Next buttons navigate sequentially through lessons across section boundaries. Sections are flattened into a linear list for prev/next calculation. Locked or non-viewable lessons are skipped (buttons disabled).

Locked Lessons

If a lesson is locked (is_locked: true), the screen shows a lock icon with "This lesson is locked" message and a "Back to Course" button.

7. Enrollment

Enrollment Flow

  1. User taps "Enroll in Course" on the course detail screen
  2. Optimistic update: set isEnrolled: true, progress: 0
  3. API call: enrollInCourse(courseId)
  4. Server returns CourseEnrollResponse with updated CourseTrack
  5. Refetch course detail to unlock lesson content
  6. UI switches to "Continue Learning" button + progress bar

Lock Screens

Non-enrolled users of private/restricted courses see a lock screen, similar to spaces. The CourseLockscreenConfig supports custom blocks, join requests, and external redirects.

8. Progress Tracking

Completion Flow

  1. User taps "Mark as Complete" on a lesson
  2. Optimistic update: add lesson ID to completed_lessons array
  3. API call: toggleLessonCompletion(courseId, lessonId, 'completed')
  4. Server recalculates progress percentage
  5. Response includes updated CourseTrack
  6. If is_completed: true, shows "Course Complete!" alert
  7. Animated success badge reveals, then Previous/Next buttons slide in
Optimistic updates revert on failure. If the API call fails, the lesson is removed from completed_lessons and progress reverts. The server's CourseTrack is always authoritative.

Progress Bar

components/course/ProgressBar.tsx renders a horizontal progress bar:

  • Color: primary until 99%, then success (green) at 100%
  • Clamped to 0–100 range
  • Optional percentage label
  • Configurable height (default 6px)

9. Lesson Content Rendering

HTML Content

Lesson content is stored as HTML in the content field and rendered via the HtmlContent component (see Components).

Hero Image Extraction

The first <figure> containing an <img> (or standalone <img>) is extracted from the HTML and rendered as a full-width hero image above the body. This prevents the image from appearing twice.

Lazy Loading

The course detail API may return lessons with empty content fields when lazy_load is enabled. In this case, the lesson view fetches individual lesson content via getLessonBySlug().

Intended lesson slug. When navigating to a lesson via deep link, the course is fetched with intendedLessonSlug query parameter. The server pre-populates that specific lesson's content, avoiding the lazy-load second fetch.

Video & Documents

  • YouTube: If lesson.meta.media.url contains a YouTube URL, a YouTubeEmbed component renders below the hero
  • Documents: lesson.meta.document_lists array rendered as downloadable file cards with file-type icons (PDF, image, video, audio, archive). Tapping opens via Linking.openURL()

Comments

Lesson comments are enabled when lesson.meta.enable_comments !== 'no' AND course.settings.disable_comments !== 'yes'. The comments button navigates to /comments/[lessonId] using the lesson's ID as the post ID.

11. Home Widget

components/home/CoursesWidget.tsx shows a horizontal carousel on the home screen:

  • Fetches enrolled courses: getCourses({ type: 'enrolled', per_page: 5 })
  • Each card: cover photo + gradient + title + progress bar
  • Stale time: 120 seconds (only refetches if older)
  • If no enrollments: shows "Browse Courses" CTA button
  • Tapping a card navigates to course detail

13. Rules

  1. Use course slug for fetching, course ID for mutations. getCourseBySlug(slug) and getLessonBySlug() use slugs. enrollInCourse(id) and toggleLessonCompletion(id, lessonId) use numeric IDs. Don't mix them up.
  2. Pass intendedLessonSlug for deep link navigation. When navigating to a specific lesson, include this parameter in the course detail fetch. The server pre-populates that lesson's content, avoiding a second fetch.
  3. Handle lazy-loaded lesson content. Course detail may return lessons with empty content fields. Always check for empty content and fetch individually via getLessonBySlug() when needed.
  4. Optimistic updates are authoritative until server responds. Add lesson ID to completed_lessons immediately. On API success, replace the entire track with the server response. On failure, revert the optimistic change.
  5. Flatten sections for prev/next navigation. Lessons span multiple sections but prev/next navigation is linear. Flatten all sections' lesson arrays into a single list sorted by section serial then lesson serial.
  6. Check both lesson and course settings for comments. Comments are enabled only when lesson.meta.enable_comments !== 'no' AND course.settings.disable_comments !== 'yes'. Either flag can disable them.
  7. Hero images are extracted from HTML, not duplicated. The first <img> or <figure> is removed from the HTML body after being rendered as a hero. Don't render the full HTML unmodified or the image appears twice.
  8. Progress bar turns green at 100%. Use primary color for 0–99% and success for 100%. This is the ProgressBar component's built-in behavior — don't override it.
  9. Cache course detail by slug. Use cache key tbc_course_{slug} with useCachedData. The home widget uses its own cache key with a 120-second stale time.
  10. Lesson IDs in completed_lessons are strings. The server returns lesson IDs as strings in the array, not numbers. Compare with String(lesson.id) when checking completion status.
