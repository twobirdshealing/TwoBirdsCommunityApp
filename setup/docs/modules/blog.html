API Endpoints

All endpoints use the standard WordPress REST API. The module’s services/blogApi.ts wraps these with the wpRequest() helper.

MethodEndpointDescription
GET/wp-json/wp/v2/postsList blog posts (paginated)
GET/wp-json/wp/v2/posts/:idSingle blog post
GET/wp-json/wp/v2/comments?post=:idComments for a post

Setup

1. Enable the module

Register the blog module in modules/_registry.ts:

import { blogModule } from './blog/module';

// Add to the modules array:
blogModule,

2. Add route stubs

Create the route stub files listed above in app/blog/ and app/blog-comments/. Each is a single re-export line pointing to the corresponding screen.

3. WordPress configuration

No server-side setup required. The Blog module reads from the built-in WordPress REST API that is enabled by default on all WordPress sites.

If your site has the REST API disabled or restricted, ensure that /wp-json/wp/v2/posts and /wp-json/wp/v2/comments are accessible.
