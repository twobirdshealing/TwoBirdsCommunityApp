API Endpoints

All endpoints require authentication and use the API base /wp-json/tbc-bc/v1. The module’s services/bookclubApi.ts uses the authenticated API client.

MethodEndpointDescription
GETtbc-bc/v1/booksList all books
GETtbc-bc/v1/books/:idSingle book with chapters
GETtbc-bc/v1/books/:id/progressUser’s listening progress
POSTtbc-bc/v1/books/:id/progressSave listening progress
GETtbc-bc/v1/books/:id/bookmarksUser’s bookmarks for a book
POSTtbc-bc/v1/books/:id/bookmarksCreate a bookmark
DELETEtbc-bc/v1/bookmarks/:idDelete a bookmark
GETtbc-bc/v1/meetingsUpcoming meeting schedule

Features

Audiobook Player

  • Chapter navigation — browse and jump between chapters
  • Progress tracking — automatically saves and resumes playback position per user
  • Bookmarks — save timestamped bookmarks within chapters
  • Speed control — adjustable playback speed

Persistent Mini Player

The MiniPlayer component is rendered above the tab bar via the tabBarAddon registration. When audio is playing, a compact player with play/pause and progress appears on every screen, allowing users to control playback without navigating back to the player.

Audio Provider

The AudioPlayerProvider context wraps the entire app (registered as a module provider with order 50). This gives all screens access to the global playback state — current book, chapter, position, and play/pause controls.

Meetings

The meeting schedule shows upcoming book club meetings managed through the companion plugin. Users can view dates, times, and meeting details via the ScheduleSheet bottom sheet.

Setup

1. Install the companion plugin

Upload and activate the tbc-book-club plugin on your WordPress site (found in companion plugins/tbc-book-club/).

2. Configure books

In WordPress admin, use the Book Club settings to add books, upload audio chapters, and schedule meetings.

3. Enable the module

Register the bookclub module in modules/_registry.ts:

import { bookclubModule } from './bookclub/module';

// Add to the modules array:
bookclubModule,

4. Add route stubs

Create the route stub files listed above in app/bookclub/. Each is a single re-export line pointing to the corresponding screen.

Dev client rebuild required. The Book Club module uses expo-av for audio playback. If your dev client doesn’t already include this native module, you’ll need to run a new EAS dev client build after enabling the module.
