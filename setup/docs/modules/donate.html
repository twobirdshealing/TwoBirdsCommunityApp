Overview

The Donate module adds a heart icon to the tab bar that opens a donation page in a WebView. It does not render a screen — instead, tapping the tab intercepts navigation and opens the configured URL. The module also registers a matching menu item in the user menu for the same action.

Setup

1. Configure the donation URL

In modules/donate/module.tsx, update the DONATE_ROUTE_PARAMS object with the path to your donation page:

const DONATE_ROUTE_PARAMS = {
  url: `${SITE_URL}/calendar/donate/`,  // SETUP: Path to your donation page
  title: 'Donate',
  rightIcon: 'cart-outline',
  rightAction: 'cart',
};

2. Enable the module

In modules/_registry.ts, ensure the donate module is registered:

import { donateModule } from './donate/module';

export const modules = [
  donateModule,
  // ...other modules
];

3. Add the route stub

Create (or verify) the tab route stub at app/(tabs)/donate.tsx. Since interceptPress prevents navigation, this component never actually renders — it returns an empty View:

import { View } from 'react-native';
export default () => <View />;
Why the stub exists: Expo Router requires a file in app/(tabs)/ for every tab. Even though the Donate tab never navigates to this screen (the press is intercepted), the route file must exist or the tab registration will fail.

How It Works

When the user taps the Donate tab:

  1. The tab bar calls the tab's interceptPress function instead of navigating
  2. interceptPress uses the router to push /webview with the donation page URL and title
  3. The WebView screen opens with the donation page, including a cart icon in the header

The same URL is opened when the user taps the "Donate" menu item in the user menu.

Module Manifest

The manifest in modules/donate/module.tsx:

export const donateModule: ModuleManifest = {
  id: 'donate',
  name: 'Donate',
  version: '1.0.0',

  tab: {
    name: 'donate',
    title: 'Donate',
    icon: 'heart',
    iconOutline: 'heart-outline',
    order: 100,
    component: () => null,
    hideMenuKey: 'donate',
    tabColor: 'error',
    tabBarIcon: ({ focused, color }) => (
      <DonateTabIcon focused={focused} color={color} />
    ),
    interceptPress: (router) => {
      router.push({ pathname: '/webview', params: DONATE_ROUTE_PARAMS });
    },
  },

  menuItems: [
    {
      id: 'donate',
      label: 'Donate',
      icon: 'heart-outline',
      route: { pathname: '/webview', params: DONATE_ROUTE_PARAMS },
      order: 60,
      hideMenuKey: 'donate',
      iconColor: 'error',
    },
  ],
};

The "Tab as Link" Pattern

This module demonstrates a useful pattern: using a tab as a link rather than a screen. The tab icon appears in the tab bar like any other tab, but tapping it opens a WebView instead of navigating to a screen component.

Key pieces that make this work:

  1. interceptPress — a function on the tab registration that receives the router. When present, the tab bar calls this function instead of performing its default navigation.
  2. component: () => null — the tab component returns null since it never actually renders.
  3. Route stub still required — Expo Router needs a file at app/(tabs)/donate.tsx even though the screen is never shown.
When to use this pattern: Any time you want a tab bar shortcut that opens external content (WebView, deep link, modal) instead of an in-app screen. Common examples: donation pages, external stores, support chat.

Animated Tab Icon

The DonateTabIcon component replaces the standard tab icon with a pulsing heart animation. Combined with tabColor: 'error', the icon renders in red (the theme's error color) to draw attention.

The custom icon is provided via the tabBarIcon property on the tab registration, which receives focused and color props from the tab bar.
