Access

The YouTube module does not register a tab or menu item. Users access it through the home screen widget and its “See All” link.

API Endpoints

All endpoints are public (no authentication required). The module’s services/youtubeApi.ts uses fetch() directly rather than the authenticated API client.

MethodEndpointDescription
GETtbc-yt/v1/latest?limit=NLatest videos from the channel
GETtbc-yt/v1/playlistsAll playlists for the channel
GETtbc-yt/v1/playlists/:id/videosVideos in a specific playlist
GETtbc-yt/v1/configModule config (channel URL for Subscribe button)
Public endpoints. These endpoints do not require authentication. The tbc-youtube plugin caches YouTube API responses server-side, so the app fetches cached data from your WordPress site rather than hitting the YouTube Data API directly.

Setup

1. Install the companion plugin

Upload and activate the tbc-youtube plugin on your WordPress site (found in companion plugins/tbc-youtube/).

2. Configure the plugin

In WordPress admin, go to the tbc-youtube settings and enter:

  • YouTube Data API v3 key — obtain from the Google Cloud Console
  • Channel ID — your YouTube channel ID

The plugin handles server-side caching automatically.

3. Enable the module

Register the youtube module in modules/_registry.ts:

import { youtubeModule } from './youtube/module';

// Add to the modules array:
youtubeModule,

4. Add route stubs

Create the route stub files listed above in app/youtube/. Each is a single re-export line pointing to the corresponding screen.
