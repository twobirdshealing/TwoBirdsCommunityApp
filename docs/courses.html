1. Overview

Courses are structured learning content managed through Fluent Community's LMS. The app provides:

  • Course browsing — all courses and enrolled courses tabs with search
  • Category filtering — horizontal chip bar to filter by course topic
  • Course detail — sections with collapsible lesson lists, progress tracking
  • Lesson viewer — HTML content, YouTube videos, documents, comments
  • Lock screens — custom lock screens for private/restricted courses with join-request support
  • Quizzes — full quiz lesson type with multiple choice, scoring, pass/fail, retake
  • Media players — YouTube, Vimeo, Wistia, and other oEmbed providers
  • Progress — per-lesson completion with optimistic updates
  • Completion celebration — confetti modal when all lessons are completed
  • Resource links — clickable external links configured in course settings
  • Free preview — non-enrolled users can preview select lessons
  • Drip content — scheduled unlock dates shown on locked sections/lessons
  • Home widget — carousel of enrolled courses on the home screen

Navigation Flow

Courses List (/courses)
  → Course Detail (/courses/[slug])
    → Lesson View (/courses/[slug]/lesson/[lessonSlug])
      → Previous / Next lesson navigation

  title: string;

CourseTrack (Progress)

interface CourseTrack {
  completed_lessons: string[];   // Array of lesson IDs (as strings)

}

4. Course List Screen

app/courses/index.tsx — route: /courses

  • Tabs: "All Courses" and "My Courses" (enrolled only)
  • Search: Text input with 400ms debounce
  • Category filtering: Horizontal chip bar fetched with with_categories: 1. Tapping a chip re-fetches with topic_slug. "All" chip clears the filter.
  • Search optimization: Uses a ref-based approach to prevent duplicate API calls — fetchCourses reads search from a ref rather than closing over the state, so useCallback identity doesn't change on every keystroke.
  • List: FlashList with infinite scroll pagination
  • Pull-to-refresh: Resets pagination and refetches
  • Cards: CourseCard with cover photo, title, stats, and progress bar for enrolled courses

5. Course Detail Screen

app/courses/[slug].tsx — route: /courses/[slug]

Layout

  1. Hero section — cover photo (or placeholder icon) with emoji badge and stats overlay (sections, lessons, students)
  2. Instructor card — uses unified Avatar + UserDisplayName components with verification badge support. Tappable to profile. Shows short_description bio (2 lines). Conditionally hidden via hide_instructor_view.
  3. Enrollment CTA — "Enroll in Course" button, or progress bar + "Continue Learning" if enrolled
  4. Resource links — clickable links from course.settings.links. Filters out disabled links (enabled !== 'no'). Opens via Linking.openURL(). Shows link icon + title + open-in-new icon.
  5. Course details — rich text description section
  6. Sections list — collapsible sections with lesson rows showing lock/check/number indicators
  7. Lock screen — non-enrolled users see CourseLockScreen component when course is private/restricted. Supports custom HTML lock content, join-request button, and pending request state.

Continue Learning

The "Continue Learning" button navigates to the first incomplete lesson. If all lessons are complete, it navigates to the first lesson in the course.

6. Lesson View Screen

app/courses/[slug]/lesson/[lessonSlug].tsx — route: /courses/[slug]/lesson/[lessonSlug]

Layout

  1. Hero image — extracted from the first <img> or <figure> in the lesson HTML (removed from body to avoid duplication)
  2. YouTube video — if lesson.meta.media.url contains a YouTube URL, renders YouTubeEmbed
  3. oEmbed video — if media URL is not YouTube, falls back to OEmbedPlayer WebView using lesson.meta.media.html
  4. Inline CSS — lesson's inline_css prepended as <style> tag to content HTML
  5. Quiz — if content_type === 'quiz', renders QuizView instead of HTML content
  6. Lesson body — remaining HTML rendered via HtmlContent component
  7. Access message — locked lessons show lesson.access_message HTML instead of hardcoded text
  8. Documents — downloadable files from lesson.meta.document_lists
  9. Comments — if enabled, shows comment count badge and links to comments screen
  10. Sticky bottom bar — state-specific UI (see below)

Bottom Bar States

StateUI
Not enrolledPrevious / Next navigation buttons
Enrolled, not complete"Mark as Complete" button
CompletingLoading spinner + success animation
CompletedSuccess badge + Previous / Next navigation
Quiz not passed (enforced)Server blocks completion API, returns error if quiz score below passing score

Completion

When a lesson is marked complete and is_completed: true is returned in the response (meaning the entire course is now complete), the app triggers the CompletionCelebration modal with confetti and stats instead of a plain alert.

Lesson Navigation

Previous/Next buttons navigate sequentially through lessons across section boundaries. Sections are flattened into a linear list for prev/next calculation. Locked or non-viewable lessons are skipped (buttons disabled).

Locked Lessons

If a lesson is locked (is_locked: true), the screen shows a lock icon with the lesson's access_message HTML (or a fallback "This lesson is locked" message) and a "Back to Course" button.

7. Enrollment

Enrollment Flow

  1. User taps "Enroll in Course" on the course detail screen
  2. Optimistic update: set isEnrolled: true, progress: 0
  3. API call: enrollInCourse(courseId)
  4. Server returns CourseEnrollResponse with updated CourseTrack
  5. Refetch course detail to unlock lesson content
  6. UI switches to "Continue Learning" button + progress bar

Lock Screens

Non-enrolled users of private/restricted courses see a lock screen instead of the course content. This is handled by the CourseLockScreen component (components/course/CourseLockScreen.tsx), which follows the same pattern as SpaceLockScreen:

  • Renders custom lockscreen_config.lockScreen HTML via HtmlContent when available
  • Shows "Request Access" button when can_request_join and not already pending
  • Shows "Pending" badge when a join request is already submitted
  • Falls back to a default lock card with lock icon + title + description when no custom lock screen HTML is configured

8. Progress Tracking

Completion Flow

  1. User taps "Mark as Complete" on a lesson
  2. Optimistic update: add lesson ID to completed_lessons array
  3. API call: toggleLessonCompletion(courseId, lessonId, 'completed')
  4. Server recalculates progress percentage
  5. Response includes updated CourseTrack
  6. If is_completed: true, triggers CompletionCelebration modal (confetti + stats)
  7. Animated success badge reveals, then Previous/Next buttons slide in
Optimistic updates revert on failure. If the API call fails, the lesson is removed from completed_lessons and progress reverts. The server's CourseTrack is always authoritative.

Progress Bar

components/course/ProgressBar.tsx renders a horizontal progress bar:

  • Color: primary until 99%, then success (green) at 100%
  • Clamped to 0–100 range
  • Optional percentage label
  • Configurable height (default 6px)

9. Lesson Content Rendering

HTML Content

Lesson content is stored as HTML in the content field and rendered via the HtmlContent component (see Components).

Hero Image Extraction

The first <figure> containing an <img> (or standalone <img>) is extracted from the HTML and rendered as a full-width hero image above the body. This prevents the image from appearing twice.

Lazy Loading

The course detail API may return lessons with empty content fields when lazy_load is enabled. In this case, the lesson view fetches individual lesson content via getLessonBySlug().

Intended lesson slug. When navigating to a lesson via deep link, the course is fetched with intendedLessonSlug query parameter. The server pre-populates that specific lesson's content, avoiding the lazy-load second fetch.

Video & Documents

  • YouTube: If lesson.meta.media.url contains a YouTube URL, a YouTubeEmbed component renders below the hero
  • oEmbed: If the media URL is not YouTube but lesson.meta.media.html exists, the OEmbedPlayer WebView renders the oEmbed HTML (supports Vimeo, Wistia, and any provider WordPress returns)
  • Documents: lesson.meta.document_lists array rendered as downloadable file cards with file-type icons (PDF, image, video, audio, archive). Tapping opens via Linking.openURL()

Comments

Lesson comments are enabled when lesson.meta.enable_comments !== 'no' AND course.settings.disable_comments !== 'yes'. The comments button navigates to /comments/[lessonId] using the lesson's ID as the post ID.

10. Quiz System

components/course/QuizView.tsx handles the full quiz-taking experience for lessons with content_type === 'quiz'.

Flow

  1. Mount — fetches existing quiz result via getQuizResult() (for retakes or resuming)
  2. Show form or results — if a previous result exists, shows the results view; otherwise shows the question form
  3. Answer questions — user selects answers for each question
  4. Submit — POST answers to server via submitQuiz(), receives score + pass/fail
  5. Results — displays score circle, pass/fail status, attempt count, and per-question breakdown

Question Types

  • single_choice — radio-style selection, only one answer allowed
  • multiple_choice — checkbox-style selection, toggle for multi-select

Only questions with question.enabled === true are shown to the user.

Answer Selection

  • Haptic feedback on each selection
  • Toggle behavior for multi-select questions
  • Validation: all questions must be answered before submit is enabled

Results View

  • Score circle showing percentage
  • Pass/fail status indicator
  • Attempt count from result.meta.attempts
  • Per-question breakdown showing correct/incorrect (unless hide_result is enabled)

Retake

The retake button clears all answers and the existing result, allowing the user to re-submit the quiz.

Server Enforcement

Quiz enforcement is server-side. The Fluent Quiz Pro module's QuizModule.php hooks into is_allowed_to_complete_lesson. If enforce_passing_score is on and the quiz hasn't been passed, the completion API rejects the request. No client-side enforcement is needed — the server is authoritative.

Quiz Settings

SettingDescription
enable_passing_scoreEnable a minimum passing score for the quiz
enforce_passing_scoreBlock lesson completion if quiz not passed
passing_scoreMinimum score percentage required to pass
hide_resultHide per-question breakdown from results view

11. Media Players

Lessons can include video from multiple providers. The detection logic determines which player to render:

  1. Check for YouTube ID — if the media URL contains a YouTube video ID (extracted via regex), render YouTubeEmbed
  2. Check for oEmbed HTML — if no YouTube ID but lesson.meta.media.html exists, render OEmbedPlayer
  3. No video — if neither condition is met, no video player is rendered

YouTubeEmbed

Existing YouTube player component. Extracts the video ID from the URL and renders an embedded YouTube player.

OEmbedPlayer

components/media/OEmbedPlayer.tsx — a WebView wrapper that renders lesson.meta.media.html in a responsive 16:9 container. Provider-agnostic: works with any oEmbed HTML WordPress returns, including Vimeo, Wistia, and other oEmbed providers.

Never render both players. The detection is sequential: YouTube ID first, then oEmbed fallback. Only one player renders per lesson.

12. Completion Celebration

components/course/CompletionCelebration.tsx renders a celebratory modal when the entire course is completed.

  • Trigger: fires when is_completed: true is returned in the completion API response
  • Trophy icon on top
  • "Course Complete!" title text
  • Total lessons stat displayed
  • Dismiss button to close the modal

Confetti Animation

50 animated particles using Reanimated (withTiming, spring physics). Uses theme tokens: themeColors.overlay for backdrop, themeColors.textInverse for the icon.

13. Category Filtering

The course list screen includes a horizontal chip bar for filtering courses by topic category. Categories are fetched alongside courses by passing with_categories: 1 to the API.

  • Tapping a category chip re-fetches courses with the topic_slug parameter
  • The "All" chip clears the filter and shows all courses
  • Chips are displayed in a horizontally scrollable row above the course list

14. Free Preview & Drip Content

Free Preview

Lessons with meta.free_preview_lesson === 'yes' are viewable by non-enrolled users. The LessonRow component shows a "Preview" badge with an eye icon for these lessons.

Drip Content

Sections and lessons can have scheduled unlock dates. When locked with a future date:

  • Locked sections show Unlocks {date} via formatMediumDate()
  • Locked lessons show Unlocks {date} badge
  • Uses section.unlock_date for sections and lesson.unclock_date for lessons (API typo preserved)

15. Resource Links

Courses can include external resource links configured in course.settings.links. These appear on the course detail screen between the enrollment CTA and the course description.

  • Disabled links (enabled !== 'no') are filtered out
  • Each link shows a link icon + title + open-in-new icon
  • Tapping opens the link's permalink via Linking.openURL()

19. Rules

  1. Use course slug for fetching, course ID for mutations. getCourseBySlug(slug) and getLessonBySlug() use slugs. enrollInCourse(id) and toggleLessonCompletion(id, lessonId) use numeric IDs. Don't mix them up.
  2. Pass intendedLessonSlug for deep link navigation. When navigating to a specific lesson, include this parameter in the course detail fetch. The server pre-populates that lesson's content, avoiding a second fetch.
  3. Handle lazy-loaded lesson content. Course detail may return lessons with empty content fields. Always check for empty content and fetch individually via getLessonBySlug() when needed.
  4. Optimistic updates are authoritative until server responds. Add lesson ID to completed_lessons immediately. On API success, replace the entire track with the server response. On failure, revert the optimistic change.
  5. Flatten sections for prev/next navigation. Lessons span multiple sections but prev/next navigation is linear. Flatten all sections' lesson arrays into a single list sorted by section serial then lesson serial.
  6. Check both lesson and course settings for comments. Comments are enabled only when lesson.meta.enable_comments !== 'no' AND course.settings.disable_comments !== 'yes'. Either flag can disable them.
  7. Hero images are extracted from HTML, not duplicated. The first <img> or <figure> is removed from the HTML body after being rendered as a hero. Don't render the full HTML unmodified or the image appears twice.
  8. Progress bar turns green at 100%. Use primary color for 0–99% and success for 100%. This is the ProgressBar component's built-in behavior — don't override it.
  9. Cache course detail by slug. Use cache key tbc_course_{slug} with useCachedData. The home widget uses its own cache key with a 120-second stale time.
  10. Lesson IDs in completed_lessons are strings. The server returns lesson IDs as strings in the array, not numbers. Compare with String(lesson.id) when checking completion status.
  11. Quiz enforcement is server-side. The Fluent Quiz Pro module hooks is_allowed_to_complete_lesson. If enforce_passing_score is on and the quiz hasn't been passed, the completion API rejects. Don't add client-side enforcement — let the server be authoritative.
  12. Use permalink for course links, not url. The API sends permalink as the URL field. Filter out disabled links (enabled !== 'no').
  13. Use unified Avatar + UserDisplayName for instructor. Don't use raw Image + Text for the instructor row — use the shared Avatar and UserDisplayName components to get verification badges and consistent styling.
  14. oEmbed fallback for non-YouTube video. Check for YouTube ID first, then fall back to OEmbedPlayer if media.html exists. Never render both.
