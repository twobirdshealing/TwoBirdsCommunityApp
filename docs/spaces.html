Spaces (Groups)

Discovery, membership, feeds, lock screens, and space groups

Key files: services/api/spaces.ts (API), app/(tabs)/spaces.tsx (discovery tab), app/space/[slug]/index.tsx (detail screen)
Spaces are Fluent Community's groups. Users discover them on the Spaces tab, join/leave, and view per-space feeds. Spaces are organized into Space Groups (categories) and support public, private, and secret privacy levels.

1. Overview

Spaces are the main organizational unit in Fluent Community. They function like groups or channels — each space has its own feed, member list, and privacy settings.

Key Concepts

ConceptDescription
SpaceA group with its own feed, members, and settings
Space GroupA category that organizes multiple spaces (displayed as tabs)
PrivacyPublic (open), Private (request to join), Secret (invite only)
RoleAdmin, Moderator, or Member within a space
Lock ScreenCustom content shown to non-members of private/secret spaces

2. Data Types

types/space.ts defines the core types:

Space

interface Space {
  id: number;
  parent_id: number | null;        // Links to SpaceGroup
  title: string;
  slug: string;
  logo: string | null;
  cover_photo: string | null;
  description: string | null;
  description_rendered?: string;
  type: 'community' | 'discussion' | 'announcement';
  privacy: 'public' | 'private' | 'secret';
  status: 'published' | 'draft' | 'archived';
  serial: string | number;          // Sort order within group
  settings: SpaceSettings;

  // Stats
  members_count?: number;
  posts_count?: number;

  // User's relationship to this space
  is_member?: boolean;
  role?: 'member' | 'moderator' | 'admin';
  pivot?: { role, status, user_id, space_id, created_at };

  // Detail endpoint only
  permissions?: Record<string, boolean>;
  membership?: { ID, display_name, pivot: { role, status } };
  lockscreen_config?: LockScreenConfig;
}

4. Discovery & Browsing

app/(tabs)/spaces.tsx is the Spaces tab. It loads all discoverable spaces and organizes them by group.

Data Loading

Two parallel fetches on mount:

  1. spacesApi.getSpaceGroups({ options_only: true }) — lightweight group list for tabs
  2. spacesApi.discoverSpaces({ per_page: 100 }) — all spaces with membership info

Group Tabs

Spaces are organized into horizontal tabs by group. Each tab shows its group name and space count. Spaces without a group appear under "Other Spaces". Within each group, spaces are sorted by their serial field.

Space Cards

Each space renders as a SpaceCard with:

  • 16:9 cover image with gradient overlay
  • Space logo or emoji icon
  • Title + privacy icon + member count
  • Description snippet (2 lines max)
  • Membership badge:
    • Admin/Mod (green) — user is admin or moderator
    • Awaiting (semi-transparent) — join request pending
    • Joined (semi-transparent) — regular member
    • Join / Request button — non-member

Search

Full-text search filters by title, description, and slug across all spaces in the current tab.

5. Space Detail Screen

app/space/[slug]/index.tsx shows an individual space:

  • SpaceInfoHeader — cover photo, logo, title, description, member count
  • QuickPostBox — shown for members, or "Join to Post" for non-members
  • Feed — space-specific feed filtered by slug
  • SpaceMenu — three-dot menu with Members and Leave options

The space is fetched by slug via spacesApi.getSpaceBySlug(slug). The detail response includes permissions, membership, and lockscreen_config not present in the discovery list.

6. Join & Leave

Join Flow

Space PrivacyBehavior
PublicInstant membership. Feed access immediately.
PrivateJoin request sent. Status becomes "pending". Admin must approve.
SecretInvite-only. No join button shown. Not in discovery.
// Join a space
const response = await spacesApi.joinSpace('general');
// Public: { success: true, data: { status: 'active' } }
// Private: { success: true, data: { status: 'pending' } }

After joining, CACHE_EVENTS.SPACES is emitted to refresh the discovery tab and update badges.

Leave Flow

  1. User taps "Leave Space" in SpaceMenu
  2. Confirmation alert shown
  3. spacesApi.leaveSpace(slug) called
  4. Navigates back to Spaces tab
  5. Cache invalidated via CACHE_EVENTS.SPACES

7. Member List

app/space/[slug]/members.tsx shows all members of a space:

  • Sorting: Admins and moderators first, then members, alphabetically within each group
  • Section headers: "Leadership" for admins/mods, "Members" for the rest
  • Per member: Avatar, display name, role badge, bio, last active, follow/message buttons
  • Pagination: 20 per page with infinite scroll

Lock Screen System

components/space/SpaceLockScreen.tsx renders for non-members of private/secret spaces. Four modes:

  1. Default — "This is a private space, ask to join" with Request button
  2. Pending — "You're all set! A team member will review your request soon"
  3. Custom — WordPress-configured blocks with images, text, and buttons
  4. Redirect — Opens redirect_url in WebView, then navigates back

Lock Screen Config

interface LockScreenConfig {

  is_pending?: boolean;         // User already requested
}

9. Space Feeds

Each space has its own feed, filtered by space slug:

// Fetch space feed
feedsApi.getFeeds({ space: 'general', per_page: 20 });

// Create post in space
feedsApi.createFeed({ space: 'general', message: '...', content_type: 'markdown' });
Always use space slug, not ID. The space parameter in feed API calls is the space's URL slug (e.g., 'general'), not its numeric database ID. This matches the Fluent Community web behavior.

Sticky Posts

Space admins can sticky/pin posts. Sticky posts appear at the top of the space feed before chronological posts. The is_sticky flag on feed objects controls this.

Quick Post Box

The space detail screen shows a QuickPostBox when the user is a member. Tapping opens the composer with the space pre-selected. Non-members see a "Join Space to Post" prompt instead.

10. Space Groups

Space Groups are categories that organize multiple spaces. They're managed in WordPress admin, not the app.

Hierarchy

SpaceGroup (e.g., "Ministry Teams")
  ├── Space: Worship Team (parent_id → group.id)
  ├── Space: Youth Ministry
  └── Space: Outreach

SpaceGroup (e.g., "Community")
  ├── Space: General
  └── Space: Prayer Requests

Ungrouped → "Other Spaces" tab
  • Groups are sorted by serial field
  • Spaces within groups are also sorted by serial
  • parent_id on a Space links it to a SpaceGroup's id
  • Groups can be collapsible with settings.is_collapsible and settings.default_collapsed

13. Rules

  1. Always use space slug, not numeric ID. API calls use the slug (e.g., 'general') for join, leave, members, and feed filtering. The only exception is getSpaceById() which exists for legacy compatibility.
  2. Use discoverSpaces() for the Spaces tab, getSpaces() for the user's joined spaces. The discover endpoint returns all visible spaces with membership info. The regular endpoint only returns spaces the user has joined.
  3. Space Groups are categories, not nested spaces. parent_id on a Space points to a SpaceGroup, not another Space. There's no space-within-space nesting. Groups are purely organizational for the discovery UI.
  4. Join behavior depends on privacy. Public = instant, Private = pending approval, Secret = not shown. Check the response status ('active' vs 'pending') to show the correct UI after joining.
  5. Emit CACHE_EVENTS.SPACES after join/leave. This refreshes the discovery tab and updates membership badges. Don't rely on local state mutations alone.
  6. Lock screen config comes from the detail endpoint only. The discovery list doesn't include lockscreen_config. Always fetch via getSpaceBySlug() before rendering lock screen content.
  7. Space feeds use the same feed API with a slug filter. Don't build separate feed fetching for spaces. Use feedsApi.getFeeds({ space: slug }). The feed system handles everything else (reactions, comments, sticky posts).
  8. Member list sorts leadership first. Admins and moderators appear under "Leadership" section before regular "Members". The sort is by role priority then alphabetical within each group.
  9. Privacy icons on SpaceCard. Lock icon = secret, people icon = private, globe = public. These help users understand access level before tapping into a space.
  10. Server enforces all access control. The app doesn't pre-check permissions. If a non-member tries to access a private space's feed, the server returns the lock screen config. Don't add client-side access checks that could get out of sync.
