Overview

The OTP module adds phone number verification as a registration step. When a user submits the registration form, the server checks whether OTP is required and, if so, sends a verification code via Twilio SMS. The user must enter the code before their account is created.

Prerequisites

  • A Twilio account with SMS capability
  • The tbc-otp companion plugin installed and activated on your WordPress site
  • The OTP module enabled in modules/_registry.ts

Setup

1. Install the companion plugin

Upload companion plugins/tbc-otp to your WordPress site and activate it.

2. Configure Twilio credentials

In WordPress admin, go to the OTP settings page and enter:

  • Twilio Account SID
  • Twilio Auth Token
  • From Phone Number or Messaging Service SID

3. Enable the module

In modules/_registry.ts, ensure the OTP module is registered:

import { otpModule } from './otp/module';

export const MODULES: ModuleManifest[] = [
  otpModule,
  // ...other modules
];
Order matters. The OTP module registers at order 20 (pre-creation). If you have other registration steps, check their order values to ensure the correct sequence.

How It Works

The OTP module hooks into the registration flow as a pre-creation step. Here is the full flow:

User fills registration form ↓ App submits to registration endpoint ↓ Server returns submitResponse with: • otp_required: true • session_key: "abc123..." ↓ OTP module's shouldActivate fires: checks submitResponse.otp_required checks submitResponse.session_key ↓ OtpRegistrationStep is shown ↓ User receives SMS code via Twilio ↓ User enters code → POST tbc-otp/v1/otp/verify ↓ [code valid?] Yes → Account created, registration continues No → Error shown, user can retry or resend

Resend & Voice Fallback

If the user does not receive the SMS, they have two options:

  • Resend SMS — sends another code via SMS. Subject to a 60-second cooldown between resends.
  • Voice call — Twilio calls the phone number and reads the code aloud. Useful when SMS delivery is unreliable.

API Endpoints

All endpoints use the base path ${SITE_URL}/wp-json/tbc-otp/v1.

POST tbc-otp/v1/otp/verify

Verify the OTP code entered by the user.

ParameterTypeDescription
session_keystringSession key returned by the registration endpoint
codestringThe verification code entered by the user

POST tbc-otp/v1/otp/resend

Resend the OTP code via SMS. Subject to a 60-second cooldown between requests.

ParameterTypeDescription
session_keystringSession key returned by the registration endpoint

POST tbc-otp/v1/otp/voice

Request a voice call fallback. Twilio calls the registered phone number and reads the code aloud.

ParameterTypeDescription
session_keystringSession key returned by the registration endpoint

Configuration

WordPress Admin Settings

After installing the tbc-otp plugin, navigate to its settings page in the WordPress admin. The following fields are required:

SettingDescription
Twilio Account SIDYour Twilio account identifier (starts with AC)
Twilio Auth TokenSecret token for API authentication
From Phone NumberA Twilio phone number to send SMS from (e.g. +15551234567)
Messaging Service SIDAlternative to From Phone Number — use a Twilio Messaging Service instead
Tip: You can use either a dedicated phone number or a Messaging Service SID. If both are provided, the Messaging Service SID takes priority. A Messaging Service is recommended for production as it handles number rotation and deliverability automatically.

Module Manifest

The module manifest in modules/otp/module.ts controls activation:

  • shouldActivate — returns true when submitResponse.otp_required is truthy and submitResponse.session_key is present
  • Registration step order — 20 (pre-creation), meaning it runs before the account is finalized
