Native WooCommerce cart with bottom sheet UI

cart Module ID

tbc-cart Companion Plugin
Companion plugin required. Install the tbc-cart WordPress plugin on a WooCommerce-enabled site. The plugin provides a lightweight REST API around the WooCommerce cart and keeps the app's header badge in sync via the X-TBC-Cart-Count response header.
Requires WooCommerce. Your WordPress site must have WooCommerce installed, active, and configured with at least one payment gateway. Payments are processed by WooCommerce — this module is a cart surface only.

Provider

CartProvider is registered at order 50 and wraps the app so any screen can call useCartCount / useCartSheet.

Response Headers

The module listens for X-TBC-Cart-Count on every authenticated API response and stores it in a shared context key (cartCount). See Header Badge Sync.

Access

The cart module does not register a tab or menu item. Users access it through the cart icon in the top header, which opens a native bottom sheet.

API Endpoints

All endpoints live under the tbc-cart/v1 namespace and require authentication (permission_callback => 'is_user_logged_in').

MethodEndpointDescription
GETtbc-cart/v1/countLightweight cart item count (reads persistent-cart user meta directly — no WC session load)
GETtbc-cart/v1/cartFull cart contents, totals, coupons, and WooCommerce display settings
PATCHtbc-cart/v1/cart/items/:keyUpdate item quantity (quantity: 0 removes the item)
DELETEtbc-cart/v1/cart/items/:keyRemove a specific line item
POSTtbc-cart/v1/cart/couponsApply a coupon code ({ code: string })
DELETEtbc-cart/v1/cart/coupons/:codeRemove an applied coupon
Why two cart endpoints? /count is a hot path called on every request to keep the header badge live — it reads from the persistent cart user meta without initializing the full WooCommerce session. /cart is called only when the user opens the bottom sheet and needs full line items, totals, and coupons.

Header Badge Sync

The cart icon's badge count stays in sync automatically via an HTTP response header. On every authenticated API call, the tbc-cart plugin injects:

X-TBC-Cart-Count: 3

The module declares this header in its responseHeaders manifest entry, and the core response-header interceptor parses it and writes cartCount into the shared context. useCartCount reads from that context, so the badge updates whenever any network call completes — no polling required.

Invalid or negative values are coerced to 0 by the module's transform function.

Setup

1. Install & configure WooCommerce

  • Install WooCommerce on your WordPress site.
  • Run the WooCommerce setup wizard (store address, currency, etc.).
  • Enable at least one payment gateway under WooCommerce → Settings → Payments (Stripe, PayPal, Square, etc.).
  • Add at least one product so there's something to add to cart.

2. Install the companion plugin

Upload and activate the tbc-cart plugin on your WordPress site (found in companion plugins/tbc-cart/). If WooCommerce is missing, you'll see an admin notice prompting you to install it.

3. (Optional) Embed WooCommerce into Fluent Community

Under TBC Community App → Cart Settings, you can optionally enable:

  • Fluent Community shop frame — iframes the WC shop and checkout inside the FC portal so members stay in the community UI while browsing.
  • Mini-cart slide-out — classic WooCommerce mini-cart overlay inside the portal.

Neither of these affects the mobile app — the app always uses the REST API endpoints above.

4. Enable the module

Register the cart module in modules/_registry.ts:

import { cartModule } from './cart/module';

// Add to the modules array:
cartModule,

5. Test

Launch the app and log in. Add a product to the cart through the WordPress site (or via the shop frame if you enabled it). On the next authenticated API call, the cart icon in the app's top header should show a badge with the item count. Tapping the icon opens the cart bottom sheet.
