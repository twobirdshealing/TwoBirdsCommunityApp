2. Theme System

Colors are synced from the WordPress site's Fluent Community theme. They flow through:

Fluent Community (WordPress) → REST API (/tbc-ca/v1/theme/colors) → ThemeContext → useTheme().colors
Never hardcode colors. Always use const { colors } = useTheme() inside components. The only exceptions are: shadowColor: '#000' (iOS standard), YouTube brand red, and video player backgrounds.

These are the primary building blocks used across the entire app. All live in components/common/.

Button Core

components/common/Button.tsx

Unified button component with 4 variants, 2 sizes, loading/disabled states, and optional icon. Built on AnimatedPressable for consistent spring-scale press feedback. Extends AnimatedPressableProps so all pressable props (onPress, disabled, etc.) are accepted.

disabledbooleanfalseDisable (also auto-set when loading)
iconIonicons name—Icon left of title
iconOnlybooleanfalseCircular icon-only button (hides title)
iconColorstring—Override icon color
styleViewStyle—Override container style
textStyleTextStyle—Override text style

import { Button } from '@/components/common/Button';

// Primary (default)
<Button title="Sign In" onPress={handleLogin} loading={isLoading} />

// Secondary outline
<Button title="Following" variant="secondary" size="sm" onPress={handleFollow} />

// Text link
<Button title="Back to Login" variant="text" onPress={() => router.back()} />

// Destructive
<Button title="Delete" variant="destructive" onPress={handleDelete} />

// With icon
<Button title="Enroll" icon="add-circle-outline" onPress={handleEnroll} />

// Icon-only (circular)
<Button icon="chatbubble-outline" iconOnly variant="secondary" onPress={handleChat} />
Tip: Don't pass disabled={loading} — Button auto-disables when loading is true. Only pass disabled for additional conditions (e.g., form validation).

components/common/TextInputField.tsx

Themed form input with optional label, error state, and password toggle. For standard forms (login, registration, profile edit). Not for chat inputs, search bars, or verification code inputs. Uses forwardRef so you can pass a ref for focus management.

errorstring—Error message below input (colors border red)

containerStyleViewStyle—Override container style

AnimatedPressable Core

components/common/AnimatedPressable.tsx

Pressable wrapper with Reanimated spring-scale animation and haptic feedback. The foundation for Button and all tappable cards/widgets. Uses forwardRef.

PropTypeDefaultDescription
animatedbooleantrueEnable/disable spring animation
scaleValuenumber0.98Scale target when pressed
hapticbooleantrueFire hapticLight() on press
...restPressableProps—All React Native Pressable props
When to use: Use AnimatedPressable for tappable cards, list items, and custom interactive elements. For standard buttons, use Button instead. If you add custom haptic in onPress, pass haptic={false} to avoid double haptic feedback. 
import { AnimatedPressable } from '@/components/common/AnimatedPressable';

// Tappable card
<AnimatedPressable onPress={() => router.push(`/profile/${username}`)}>
  {/* card content */}
</AnimatedPressable>

// With custom haptic (disable built-in)
<AnimatedPressable haptic={false} onPress={() => { hapticMedium(); doAction(); }}>
  {/* ... */}
</AnimatedPressable>

10. Import Rules & Conventions

Direct Imports Only

No barrel files. Always import directly from the source file. Metro bundler doesn't tree-shake — barrel imports load every sibling module. 
// Correct
import { Avatar } from '@/components/common/Avatar';
import { Button } from '@/components/common/Button';
import { spacing, sizing, typography } from '@/constants/layout';

// WRONG (barrel import)
import { Avatar, Button } from '@/components/common';

Theme Colors

// Always use the hook
const { colors } = useTheme();

// Use tokens, not raw colors
backgroundColor: colors.primary       // correct
backgroundColor: '#6366F1'            // WRONG

// For text on colored buttons
color: colors.textInverse             // correct
color: '#fff'                         // WRONG

// For transparent variants
import { withOpacity } from '@/constants/colors';
backgroundColor: withOpacity(colors.primary, 0.12)

Component Patterns

// Button: don't duplicate disabled with loading
<Button title="Save" loading={saving} />                    // correct
<Button title="Save" loading={saving} disabled={saving} />  // redundant

// AnimatedPressable: avoid double haptic
<AnimatedPressable haptic={false} onPress={() => { hapticMedium(); doThing(); }}>
  {/* custom haptic — disable built-in hapticLight */}
</AnimatedPressable>

// TextInputField: for forms only
<TextInputField label="Name" ... />   // form input — use TextInputField
// Chat input, search bar, verification codes — use raw TextInput (specialized UX)
