2. TBC Community App

companion plugins/tbc-community-app/ Core

The main bridge plugin. Every app installation requires this. It provides JWT authentication, registration, the batch API, push notifications, app configuration delivery, and authenticated WebView sessions.

3. TBC Multi Reactions

companion plugins/tbc-multi-reactions/ Core

Replaces Fluent Community’s default single-heart reaction with a multi-emoji reaction system. Supports posts, comments, and chat messages. Reaction icons can be customized in WP admin (PNG, SVG, GIF, WEBP).

Endpoints

MethodEndpointAuthPurpose
GET/configNoGet enabled reaction types and icon URLs
POST/swapYesSwap reaction type on an existing reaction
GET/breakdown/{type}/{id}NoGet reaction counts by type for a post/comment
GET/breakdown/{type}/{id}/usersNoGet reaction breakdown with user details

How It Works

  • Injects reaction data into Fluent Community’s standard feed/comment API responses (no extra API calls needed for basic display)
  • The /config endpoint provides icon URLs so the app can display custom reaction graphics
  • The /swap endpoint lets users change their reaction type without removing and re-adding
  • Controlled by the multi-reactions app module (registered in modules/_registry.ts). When the module is not registered, core falls back to a simple like button

4. TBC Starter Theme

companion plugins/tbc-starter-theme/ Core

A lightweight WordPress theme designed for sites running Fluent Community. It stays out of the way on community portal pages (Fluent Community renders its own Vue.js SPA) while providing clean styling for regular WordPress pages.

Key Features

  • Zero interference with Fluent Community’s SPA rendering
  • Dark/light mode sync via CSS variables (bridges WordPress theme to Fluent Community colors)
  • Minimal chrome — blank-canvas approach with minimal header/footer
  • Block editor support for regular WordPress pages
  • Lightweight styling for blog posts and WooCommerce pages
This theme has no REST endpoints. It’s a visual WordPress theme only — the app doesn’t communicate with it directly. Install it on the site so non-community pages (blog, shop, etc.) look decent.

5. Add-on Plugins

Each add-on module ships with a companion WordPress plugin that provides its REST endpoints. Add-on plugins are sold separately and come with their matching app module.

How Add-on Plugins Work

  • Each add-on plugin registers its own REST namespace (e.g., tbc-yourplugin/v1)
  • Plugins hook into the core plugin's filter system (tbc_ca_registration_config, tbc_ca_pre_register, etc.) to extend functionality
  • The matching app module communicates with the plugin's REST endpoints
  • All add-on plugins are completely optional — the app adapts automatically based on which plugins are active

Setup Pattern

  1. Install the companion plugin on WordPress and activate it
  2. Configure the plugin in WP admin (API keys, content, etc.)
  3. Enable the matching app module in modules/_registry.ts
  4. Rebuild the app
Completely optional. Without any add-on plugins, the core app works fully. Add-on plugins extend functionality — when they’re not installed, the corresponding features simply don’t appear. The app discovers available add-on capabilities via the server config.

6. App-Side URL Constants

The app references plugin APIs via URL constants defined in constants/config.ts. All are derived from SITE_URL.

// Core plugin APIs
export const TBC_CA_URL  = `${SITE_URL}/wp-json/tbc-ca/v1`;       // Auth, registration, config

// Also used:
export const API_URL     = `${SITE_URL}/wp-json/fluent-community/v2`;
export const WP_REST_URL = `${SITE_URL}/wp-json/wp/v2`;

Module-specific API URLs are defined locally in each module's service files (not as global constants). Each module constructs its own base URL from SITE_URL.

Service files pass these as baseUrl to the HTTP client:

// services/api/registration.ts
const response = await fetch(`${TBC_CA_URL}/auth/register`, { ... });

// modules/yourmodule/services/yourApi.ts — module-local constant
const API_BASE = `${SITE_URL}/wp-json/your-plugin/v1`;
const response = await request('/items', { baseUrl: API_BASE });

7. Installation

For Buyers (White-Label Setup)

  1. Upload the 2 core plugins + theme to wp-content/plugins/ and wp-content/themes/
  2. Activate all plugins and the theme in WordPress admin
  3. No configuration needed for tbc-community-app — it auto-detects Fluent Community
  4. Configure tbc-multi-reactions in WP admin to customize reaction icons (optional)

For Addon Plugins

  1. Upload the plugin to wp-content/plugins/ and activate
  2. Configure in WP admin (API keys, content, etc.)
  3. Enable the matching app module in modules/_registry.ts
  4. Rebuild the app

Plugin Updates

Plugins are updated manually (upload new version via FTP or WP admin). After updating a plugin:

  • Bump the plugin version number in the main PHP file header
  • Update the changelog in the plugin
  • Clear any relevant caches on the server
Always update the plugin version number — even for small changes. WordPress caches plugin metadata, and unchanged versions may not clear old cached responses.
