22. Module Ecosystem

The app's module system is an architecture-level exclusive — Fluent Community has no equivalent. Modules are self-contained feature packages that register any combination of: bottom tabs, home widgets, menu items, header icons, context providers, and tab bar addons.

Each module gets its own dedicated documentation page. This section provides the overview.
Module Type Tab Widget Menu Description
Blog ADD-ON ✕ ✓ ✓ WordPress blog integration with post listing, full content view, and WP comment system
YouTube ADD-ON ✕ ✓ ✕ YouTube channel videos and playlists with server-side caching. Companion plugin: tbc-youtube
Book Club ADD-ON ✕ ✓ ✓ Audiobook library with full player, chapter navigation, bookmarks, meeting schedule, persistent mini player. Companion plugin: tbc-book-club
Calendar Site-specific ✓ ✓ ✕ Month grid calendar with event listing, WooCommerce booking integration, RSVP/waitlist. Two home widgets. Companion plugin: tbc-calendar-fluent
Donate Site-specific ✓ ✕ ✓ Animated heart tab icon that opens WooCommerce donation page in authenticated WebView
Donor Dashboard Site-specific ✕ ✕ ✓ Menu item that opens donor dashboard in authenticated WebView

Module Architecture Highlights

  • Zero-touch core: Modules never modify core files. Register once in _registry.ts and the system wires everything up.
  • Server-driven visibility: The server's hide_menu array can hide module tabs and menu items without an app update.
  • Forward-compatible widgets: When a new module is enabled, its widget auto-appears on the home screen in the next launch.
  • Push notification routing: Modules can register push handlers to intercept and route their own notification types.
  • Context providers: Modules can inject React context providers (e.g., Book Club's audio player state) into the app's provider tree.
  • Tab bar addons: Modules can inject persistent UI above the tab bar (e.g., Book Club's mini player).

  • $499 one-time, updates included — buy once, own the code, get future snapshots. No subscription, no renewal fees, no domain limits.
  • Self-publishing — you publish to your own App Store and Play Store accounts. Full control over release timing, listing content, and update schedule. No waiting on a vendor to push your build.
  • No vendor lock-in — if we stop existing, your code still runs. You can maintain and extend it independently.
  • Companion WordPress plugins included — server-side code ships with the app. You control both sides of the stack.
  • Add-on module ecosystem — buy only the features you need. Blog, YouTube, Book Club available separately with companion WordPress plugins.

Native App Quality

  • Native multi-step registration — OTP verification via SMS (Twilio) with voice call fallback. Password strength meter. Profile completion wizard. Not a WebView form.
  • JWT with silent refresh — tokens in OS secure storage (Keychain / Keystore). Automatic re-authentication on expiry. No login prompts during normal use.
  • Real-time WebSocket messaging — Pusher integration with private channels. Messages appear instantly, not on a polling interval.
  • Deep linking throughout — push notifications route to the exact screen. Web URLs map to app routes. Custom URL scheme for cross-app linking.
  • Rich text editor — native markdown toolbar (bold, italic, links, images, code blocks), not a basic text input.
  • Multi-reaction system — custom emoji reactions with breakdown view. Companion plugin handles server-side. Not just a like button.
  • Haptic feedback — tactile response on interactions. Small detail, big polish.

Extensibility

  • Module system — add features without touching core code. Each module can register tabs, widgets, menu items, header icons, context providers, and tab bar addons.
  • Add-on marketplace — Blog, YouTube channel integration (with server-side caching plugin), Book Club (audiobook player with chapters, bookmarks, mini player).
  • Build your own modules — the module system is documented and designed for third-party development.
  • Companion plugin architecture — each feature has matching WordPress plugin(s). You control server and client.

Developer Experience

  • Expo + EAS — industry-standard build pipeline. Over-the-air updates possible. No CI/CD setup needed.
  • Tagged logger system — zero-cost in production, structured debug output in development.
  • Batch API — single startup request replaces ~11 individual calls. Fast cold start.
  • Comprehensive developer documentation — architecture, components, theme system, module system, setup guide, all in structured HTML docs.
