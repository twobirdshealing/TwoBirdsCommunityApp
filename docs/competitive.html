Key Differentiators (Our Advantages)

You own the code. This is the single biggest difference. With our app, you receive the full React Native (Expo) source code. You can modify any screen, add any feature, integrate any third-party service, or hire any React Native developer to extend it. With the competitor, you get a managed service — your app is built and deployed by their platform. If they shut down, raise prices, or don't add a feature you need, you're stuck.

Architecture & Ownership

  • Full source code — React Native (Expo), one of the most widely-used mobile frameworks. Massive developer ecosystem. Any React Native developer can work on it.
  • $499 one-time, updates included — buy once, own the code, get future snapshots. No subscription, no renewal fees, no domain limits.
  • Self-publishing — you publish to your own App Store and Play Store accounts. Full control over release timing, listing content, and update schedule. No waiting on a vendor to push your build.
  • No vendor lock-in — if we stop existing, your code still runs. You can maintain and extend it independently.
  • Companion WordPress plugins included — server-side code ships with the app. You control both sides of the stack.
  • Add-on module ecosystem — buy only the features you need. Blog, YouTube, Book Club available separately with companion WordPress plugins.

Native App Quality

  • Native multi-step registration — OTP verification via SMS (Twilio) with voice call fallback. Password strength meter. Profile completion wizard. Not a WebView form.
  • JWT with silent refresh — tokens in OS secure storage (Keychain / Keystore). Automatic re-authentication on expiry. No login prompts during normal use.
  • Real-time WebSocket messaging — Pusher integration with private channels. Messages appear instantly, not on a polling interval.
  • Deep linking throughout — push notifications route to the exact screen. Web URLs map to app routes. Custom URL scheme for cross-app linking.
  • Rich text editor — native markdown toolbar (bold, italic, links, images, code blocks), not a basic text input.
  • Multi-reaction system — custom emoji reactions with breakdown view. Companion plugin handles server-side. Not just a like button.
  • Haptic feedback — tactile response on interactions. Small detail, big polish.

Extensibility

  • Module system — add features without touching core code. Each module can register tabs, widgets, menu items, header icons, context providers, and tab bar addons.
  • Add-on marketplace — Blog, YouTube channel integration (with server-side caching plugin), Book Club (audiobook player with chapters, bookmarks, mini player).
  • Build your own modules — the module system is documented and designed for third-party development.
  • Companion plugin architecture — each feature has matching WordPress plugin(s). You control server and client.

Setup & Maintenance

  • Local setup dashboard — full web GUI for configuration, asset management, builds, updates, and module installation. No competitor offers this. Buyers configure their app through a visual interface instead of editing JSON files.
  • One-click updates — enter license key, click update. Automatic backup before every update, automatic restore if something goes wrong. Protected files (config, assets, modules) are never overwritten. No git knowledge required.
  • Module import via dashboard — upload a .tar.gz module package, dashboard handles extraction, registration, and route stub creation. Add features without touching code.
  • Build management — trigger EAS builds, view history, submit to App Store Connect, download Android bundles — all from the dashboard.

Developer Experience

  • Expo + EAS — industry-standard build pipeline. Over-the-air updates possible. No CI/CD setup needed.
  • Tagged logger system — zero-cost in production, structured debug output in development.
  • Batch API — single startup request replaces ~11 individual calls. Fast cold start.
  • Comprehensive developer documentation — architecture, components, theme system, module system, setup guide, all in structured HTML docs.

Who Is This For

Our ideal buyer is a developer or agency who has a client (or their own site) running Fluent Community and wants a real native mobile app — not a WebView wrapper, not a managed service they can't control. The local setup dashboard means they don't need deep React Native knowledge to get started — configuration, branding, builds, and updates are all handled through a visual GUI. They want to own the code, publish on their own accounts, and charge their clients whatever they want for the build.

This product is for you if:

  • You're a developer or agency building mobile apps for WordPress/Fluent Community clients
  • You want full source code you can read, modify, and extend — not a black box
  • You want to publish on your own App Store and Play Store accounts
  • You're comfortable following setup documentation rather than needing hand-holding
  • You want a one-time purchase with no recurring fees, domain limits, or vendor lock-in
  • You want to add value for your clients — buy the app for $499, customize it, charge them $5k–$20k for a "custom" community app

The competitor may be a better fit if:

  • You're a non-technical site owner who doesn't want to touch code
  • You need someone else to handle builds and deployment for you
  • You're comfortable with ongoing vendor dependency and per-domain licensing
  • You prefer a managed service with 24/7 support over owning the code yourself

The bottom line

We're not a SaaS platform. We're a product for people who build things. $499 for the core app, $99 per module, updates included forever. No subscriptions, no tiers, no domain counting. Your code, your clients, your business.

Where the competitor lists features we don't have yet (group messaging, leaderboard, global search), those are already on our roadmap — and because you have the source code, you could build them yourself or wait for the next snapshot.
