Companion Plugin

TBC Community App — the WordPress plugin that powers the app’s backend: REST endpoints, auth, push notifications, and more

One plugin, shipped separately. tbc-community-app.zip is delivered alongside your core update as a standalone download. Install it on your WordPress site (running Fluent Community) via Plugins → Add New → Upload Plugin. Add-on modules each ship their own companion plugin as part of the add-on purchase — see the individual module docs for those.

1. Overview

The main bridge plugin. Every app installation requires this. It provides JWT authentication, registration, the batch API, push notifications (via Expo Push API), app configuration delivery, authenticated WebView sessions, deep linking, and response header injection.

REST namespace: tbc-ca/v1.

3. REST Endpoints

Registration

MethodEndpointAuthPurpose
GET/auth/register/fieldsNoField definitions + server config (FC native custom fields, add-on capabilities)
POST/auth/registerNoCreate user account (returns JWT). Add-ons hook tbc_ca_pre_register to add verification steps.

Authentication

MethodEndpointAuthPurpose
POST/auth/loginNoLogin → returns access + refresh tokens + user profile
POST/auth/refreshNoExchange refresh token for new access token
POST/auth/logoutYesRevoke server session
POST/password/forgotNoSend password reset email
POST/password/resetNoSet new password with reset token

App Configuration

MethodEndpointAuthPurpose
GET/app-configOptionalTheme colors, branding, feature flags, socket config, maintenance, update info, social providers, portal slug, registration capabilities, time format, crash reporting (Sentry DSN + enable flag). When authenticated: UI visibility flags + maintenance bypass.
GET/badge-definitionsNoAchievement badge definitions (slugs, labels, icons)

Batch API

MethodEndpointAuthPurpose
POST/batchYesCombine up to 20 REST requests into a single HTTP call

See Caching System → Batch Startup for how the app uses this.

Push Notifications

MethodEndpointAuthPurpose
POST/push/deviceYesRegister device token (Expo Push Token)
DELETE/push/deviceYesUnregister device token (specific token, or all for user)
GET/push/settingsYesGet user’s notification preferences (grouped by category)
POST/push/settingsYesUpdate notification preferences

Push delivery uses Expo Push API (no Firebase service account needed on the server). The push feature flag must be enabled in the Features admin tab.

Chat-message push routing (1:1 / group / space)

The fluent_messaging/after_add_message hook in push/class-hooks.php reads fcom_chat_threads.provider to decide where to deep-link the notification:

Thread providerNotification deep-link
'group'/messages/group/{thread_id} (opens group chat with title carried in the payload)
'fcom' with non-null space_idRouted by the app to /messages/space/{thread_id} based on the resolved thread record
'fcom' / NULL (DM)Legacy /messages/user/{sender_id} route — kept untouched so older app versions don’t break

This is the only companion-plugin change required for the 2.4.0 group + space chat work. Everything else (CRUD, members, admin actions, real-time) is pure mobile + Fluent Messaging native — no companion-plugin endpoints needed.

Account & WebView

MethodEndpointAuthPurpose
POST/account/deactivateYesSoft-deactivate user profile (reversible via web login)
DELETE/account/deleteYesPermanently delete user account (requires {"confirm":"DELETE"})
POST/create-web-sessionYesCreate one-time login URL for in-app WebView

Deep Link Files

MethodPathPurpose
GET/.well-known/apple-app-site-associationiOS Universal Links configuration (served via early init interception + REST fallback)
GET/.well-known/assetlinks.jsonAndroid App Links configuration (served via early init interception + REST fallback)

Deep link settings (bundle ID, package name, Apple Team ID, Android SHA256, App Store ID, smart banner toggle) are configured in the admin General tab.

4. Admin Settings

The plugin adds a settings page under WP admin with 8 tabs:

TabSettings
GeneralApp version control (min version, store URLs), maintenance mode (toggle, message, bypass roles), deep linking (bundle ID, package name, URL scheme, Apple Team ID, Android SHA256, App Store ID, smart banner), data management (delete on uninstall)
FeaturesFeature flags: dark mode, push notifications, messaging (requires Fluent Messaging), courses (requires FC Course module). Profile tabs: posts, spaces, comments.
Crash ReportingSentry DSN field + Enable Crash Reporting toggle. When enabled with a valid DSN, the mobile app initializes Sentry on cold start (after a one-time two-relaunch bootstrap) and sends all native crashes, JS exceptions, and breadcrumbs to your Sentry dashboard. DSN is validated server-side. Both fields blank = SDK stays dormant. See setup guide for the full flow.
UI VisibilityRole-based element hiding matrix (cart, blog, courses, bookmarks, directory, notification settings). Custom visibility elements for module integration.
NotificationsPush notification type management: enable/disable types, set defaults, control user visibility. Types are grouped by category.
Push Log30-day push notification statistics and recent activity log (read-only).
StatisticsDevice token counts (iOS/Android), JWT session counts (active/expired/per-user).
ToolsSession management (purge all, revoke per-user), manual push notification sender.

5. Response Headers

The plugin injects custom headers into every authenticated REST response (any namespace, not just tbc-ca/v1). The mobile app reads these via a global interceptor in services/api/client.ts. See API Client → Response Headers.

HeaderValuePurpose
X-TBC-Unread-NotificationsIntegerFluent Community unread notification count
X-TBC-Unread-MessagesIntegerFluent Messaging unread thread count
X-TBC-Maintenance0 or 1Whether user should see maintenance screen
X-TBC-Profile-Incomplete0 or 1Whether user should be gated for profile completion
X-TBC-Min-App-VersionSemver stringMinimum app version (only sent when configured)

6. REST Fields

The plugin registers custom REST fields on WP REST API responses to embed Fluent Community data without extra API calls:

  • User objects (posts via _embed): fcom_avatar, fcom_is_verified, fcom_badge_slugs
  • Comment objects: fcom_author_avatar, fcom_author_is_verified, fcom_author_slug, fcom_author_badge_slugs

7. Global Helpers & Extension Hooks

Helper Functions

The plugin exposes helper functions for use by other plugins:

  • tbc_register_push_notification($args) — Register a push notification type
  • tbc_send_push_notification($user_id, $type, $data) — Send push to a single user (synchronous)
  • tbc_send_push_to_users($user_ids, $type, $title, $body, $route, $exclude_id) — Send push to multiple users via async Action Scheduler queue (recommended for external plugins)

Extension Hooks

Add-on plugins integrate via these filters:

  • tbc_ca_pre_register — Intercept before user creation (e.g., OTP verification)
  • tbc_ca_post_register — Act after user creation (e.g., mark profile incomplete)
  • tbc_ca_registration_config — Advertise add-on capabilities in the registration config
  • tbc_ca_features_config — Modify feature flags before sending to the app
  • tbc_ca_register_push_types — Register custom push notification types
  • tbc_reg_registration_response — Attach JWT tokens to registration responses

8. App-Side URL Constants

The app references plugin APIs via URL constants defined in constants/config.ts. All are derived from SITE_URL.

// Core plugin APIs
export const TBC_CA_URL  = `${SITE_URL}/wp-json/tbc-ca/v1`;       // Auth, registration, config

// Also used:
export const API_URL     = `${SITE_URL}/wp-json/fluent-community/v2`;
export const WP_REST_URL = `${SITE_URL}/wp-json/wp/v2`;

Add-on module API URLs are defined locally in each module’s service files (not as global constants). Each module constructs its own base URL from SITE_URL.

Service files pass these as baseUrl to the HTTP client:

// services/api/registration.ts
const response = await fetch(`${TBC_CA_URL}/auth/register`, { ... });

// modules/yourmodule/services/yourApi.ts — module-local constant
const API_BASE = `${SITE_URL}/wp-json/your-plugin/v1`;
const response = await request('/items', { baseUrl: API_BASE });

9. Installation

  1. Locate tbc-community-app.zip — shipped alongside your core update as a separate download
  2. In WordPress admin, go to Plugins → Add New → Upload Plugin and upload the zip
  3. Activate the plugin — no configuration needed, it auto-detects Fluent Community

Plugin Updates

To update the plugin, upload the new version via WP admin (or FTP) and reactivate. After updating:

  • Clear any relevant caches on the server
  • Bump the plugin version number in the main PHP file header (if you’re customizing)
Always update the plugin version number — even for small changes. WordPress caches plugin metadata, and unchanged versions may not clear old cached responses.
